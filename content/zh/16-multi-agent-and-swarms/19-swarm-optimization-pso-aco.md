# 面向 LLM 的群体智能优化（PSO、ACO）

> 生物启发式优化正在 LLM 时代卷土重来。**LMPSO**（arXiv:2504.09247）把 PSO 用在 LLM 上：每个粒子的速度是一段提示词，由 LLM 生成下一个候选解，在结构化序列输出（数学表达式、程序）上效果很好。**Model Swarms**（arXiv:2410.11163）把每个 LLM 专家模型当作模型权重流形上的一个 PSO 粒子，仅用 200 个样本就在 9 个数据集上相对 12 个基线取得了 **13.3% 的平均提升**。**SwarmPrompt**（ICAART 2025）混合 PSO 与灰狼算法做提示词优化。**AMRO-S**（arXiv:2603.12933）是受 ACO 启发的信息素专家系统，用于多智能体 LLM 路由——**4.7 倍加速**、可解释的路由依据，以及把推理与学习解耦的质量门控异步更新。本课在提示词参数空间上实现 PSO、在智能体路由上实现 ACO，量化分析这些经典算法为何适配 LLM 时代，以及何时并不适用。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 09 (Parallel Swarm Networks), Phase 16 · 14 (Consensus and BFT)

## 问题背景

你有一条提示词，在任务评测上得分 62%，你想把它改得更好。最朴素的做法是无梯度的手工调整，但这种方式很难规模化。强化学习需要奖励信号和足够多的训练 rollout。对提示词做反向传播基本行不通——提示词是离散字符串，不是可微参数。

经典的生物启发式优化——面向连续搜索空间的 PSO、面向路径选择的 ACO——正是为这种场景设计的：无梯度、基于种群、单次评估成本低。把它们与 LLM 结合来完成无梯度搜索步骤，你会得到一个出人意料地实用的优化器。

同样的模式也适用于多智能体系统中的智能体*路由*。ACO 式的信息素轨迹记录了哪个智能体在哪类任务上表现最好，让路由器可以沿着轨迹进行利用（exploit），同时通过信息素衰减让路由可以被重新发现。

## 核心概念

### PSO 回顾（Kennedy & Eberhart 1995）

粒子群优化（Particle Swarm Optimization）：在连续搜索空间中维护一个粒子种群。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子自身的历史最优，`g_best` 是整个群体的最优，`w, c1, c2` 分别是惯性、认知、社会权重，`r1, r2` 是随机因子。

### PSO 用于 LLM 输出 —— LMPSO

arXiv:2504.09247 把 PSO 改造用于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一段*提示词*，描述如何把当前输出朝个体/全局最优方向修改。LLM 根据速度提示词生成新的输出。速度的「惯性」对应类似「做小幅渐进式修改」这样的提示词。

它在以下条件下效果很好：

- 输出是结构化的（可解析、可评估）。
- 适应度可以自动计算（运行测试、算术求值）。
- 种群规模小（约 10-30 个粒子），从而把 LLM 调用总量控制在可承受范围。

当适应度需要人工评审时效果就不好了——每次迭代的成本会变得难以承受。

### Model Swarms

arXiv:2410.11163 把 PSO 从输出层带到了*模型*层。每个「粒子」是一个专家 LLM（的参数）。群体通过无梯度更新把参数推向集体最优。报告结果：在 9 个数据集上相对 12 个基线取得 13.3% 的平均提升，每次迭代仅需 200 个样本。

关键洞察在于：各个 LLM 专家模型在共享参数流形上本来就彼此邻近（适配器权重、LoRA 增量）。在这个低维子空间上跑 PSO 既便宜又有效。

### ACO 回顾（Dorigo 1992）

蚁群优化（Ant Colony Optimization）：蚂蚁在图上移动；每条路径有一条信息素轨迹。蚂蚁的移动概率按信息素强度加权。完成任务的蚂蚁按解的质量成比例地沉积信息素。信息素随时间衰减。

### AMRO-S —— 用 ACO 做智能体路由

arXiv:2603.12933 用 ACO 做多智能体路由。每种任务类型是一个「目的地」，每个智能体是一条可能的路线。产出好结果的路线会得到信息素强化。主要贡献：

- **可解释的路由依据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 只有通过质量检查后才更新信息素，把推理与学习解耦。
- 在多智能体路由基准上取得 **4.7 倍加速**。

质量门控很重要：没有它，「快但错」的智能体会不断积累信息素，系统会锁死在糟糕的路由上。

### 何时对 LLM 使用 PSO / ACO

**使用 PSO 的场景：**

- 搜索空间是连续的，或可以映射为连续参数（提示词嵌入、LoRA 权重、数值生成参数）。
- 适应度计算便宜且自动。
- 种群可以很小（10-30）。

**使用 ACO 的场景：**

- 你面对的是路由或路径选择问题。
- 决策会随时间得到强化（同类任务会反复出现）。
- 你需要为路由决策提供可解释的依据。

**两者都不适用的场景：**

- 适应度需要人工评审（每次迭代成本太高）。
- 搜索空间是离散组合型的，超出了 PSO 的覆盖范围（此时改用遗传算法）。
- 实时决策有严格的延迟要求（相对单遍启发式方法，PSO/ACO 收敛较慢）。

### 为什么生物启发式方法依然能赢

基于梯度的方法需要可微信号。LLM 的输出和路由决策并不天然可微。伪梯度方法（强化学习训练的路由器、DPO 式的提示词调优器）可行，但训练成本高昂。

PSO 和 ACO 只需要一个*评估器*函数。只要你能给候选输出或路由决策打分，就能在这个空间上做优化。这让适用门槛低了很多。

### 实践中的限制

- **种群预算。** N 个粒子 × T 次迭代 × 单次评估成本。按每次 LLM 评估约 $0.02 计算，一个 20 粒子、跑 50 次迭代的 PSO 约花费 $20。要据此做预算规划。
- **探索与利用的权衡。** 信息素衰减速率与 PSO 惯性是一对权衡；衰减太快 → 遗忘已有解；太慢 → 困在早期局部最优。
- **灾难性漂移。** 如果适应度地形发生变化（新的数据分布），两种算法都可能先收敛再发散。要监控最优适应度的稳定性。

## 从零实现

`code/main.py` 实现了：

- `LMPSO` —— 在数值提示词参数（temperature、top_k 权重）上运行 PSO。每个粒子的「LLM 生成」用脚本化的适应度函数模拟。算法运行 30 次迭代，展示 g_best 的收敛过程。
- `AMRO_S` —— ACO 式路由。3 个智能体、4 种任务类型、信息素矩阵、100 个被路由的任务。打印（任务类型 → 智能体选择）分布随时间的变化，展示信息素轨迹的形成过程。
- 对比实验：在同一任务流上比较随机路由与 ACO 路由，测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：

- LMPSO：g_best 适应度在 30 次迭代内从随机水平提升到接近最优。
- AMRO-S：信息素表稳定到每种任务类型对应正确的智能体；ACO 路由在质量上比随机路由高约 30-40%，同时也降低了延迟（重试更少）。

## 交付产物

- **从小规模开始。** 10-20 个粒子、20-50 次迭代。只有当收敛曲线显示出明确收益时才扩大规模。
- **每次迭代记录信息素或 g_best。** 没有日志轨迹去调试群体优化器会非常痛苦。
- **给更新加质量门控。** 对 ACO 路由尤其如此：「快但错」的智能体绝不能积累信息素。
- **分布漂移时重置衰减。** 当评测分布发生变化时，老化的信息素已经过时；临时重置或把衰减速率加倍。
- **限制单次迭代成本。** 输出每次迭代的成本指标。一个每次迭代花 $500、只提升 0.5% 的 PSO 是无法上线的。

## 练习

1. 运行 `code/main.py`，观察 LMPSO 的收敛过程。把种群规模分别改为 5、10、20、50。收敛时间在多大规模时趋于饱和？
2. 实现一个「灾难性漂移」实验：在第 30 次迭代后更换适应度函数。PSO 的适应速度有多快？重置 `p_best` 是否有帮助？
3. 给 AMRO-S 加一个质量门控：仅当评测分数 > 0.7 时才沉积信息素。与无门控版本相比，收敛行为有何变化？
4. 阅读 LMPSO（arXiv:2504.09247）。把论文中「速度即提示词」的设计映射回你的数值速度。模拟中丢失了什么、保留了什么？
5. 阅读 AMRO-S（arXiv:2603.12933）。实现解耦的「推理快路径」配合异步信息素更新。在持续负载下，这对系统延迟有何影响？

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------------|------------------------|
| PSO | 「粒子群优化」 | Kennedy-Eberhart 1995。基于种群的无梯度优化器。 |
| ACO | 「蚁群优化」 | Dorigo 1992。通过信息素轨迹做路径/路由优化。 |
| LMPSO | 「带 LLM 生成的 PSO」 | arXiv:2504.09247。速度是一段提示词；由 LLM 生成候选解。 |
| Model Swarms | 「在专家权重上跑 PSO」 | arXiv:2410.11163。在模型参数子空间上做无梯度更新。 |
| AMRO-S | 「用 ACO 做智能体路由」 | arXiv:2603.12933。任务类型 × 智能体上的信息素矩阵。 |
| p_best / g_best | 「个体/全局最优」 | 单个粒子和整个群体迄今找到的最优解。 |
| 信息素（Pheromone） | 「路由记忆」 | 边上的强度值；随时间衰减；按质量沉积。 |
| 质量门控更新 | 「只从好的运行中学习」 | 信息素沉积以通过质量检查为前提。 |
| 灾难性漂移 | 「分布漂移」 | 适应度地形变化；旧的 p_best 和信息素失效。 |
