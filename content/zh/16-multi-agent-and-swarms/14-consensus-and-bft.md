# 智能体的共识与拜占庭容错

> 经典分布式系统的 BFT 遇上随机性的 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）用置信度探针为每张选票加权；**DecentLLMs**（arXiv:2507.14928）采用无领导者架构，工作智能体并行提出方案，再做几何中位数聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类（Hierarchical Structure Clustering）结合，把节点划分为核心（Core）和边缘（Edge）。而 "Can AI Agents Agree?"（arXiv:2603.01213）给出的实证结论很坦诚：即便是标量层面的一致性在今天也很脆弱——单个欺骗性智能体就能让 Mixture-of-Agents 失守。BFT 是必要条件，但不是充分条件。本课会构建一个最小化的 BFT 协议，注入三种智能体特有的攻击（拜占庭谎言、谄媚从众、相关错误单一文化），并测量每种共识变体的应对表现。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 07 (Society of Mind and Debate), Phase 16 · 13 (Shared Memory)
**Time:** ~75 minutes

## 问题背景

你有 N 个 LLM 智能体，各自给出一个答案。它们意见不一。多数投票选中了错误答案，因为其中两个智能体是相关的（同一基础模型、同一训练数据、同一失败模式）。第三个智能体恰好以一种全新的方式出错——于是这个多数其实是虚假多数。

现在再加一个欺骗性智能体：它故意撒谎。或者一个谄媚的智能体：它附和最后一个发言者。在经典 BFT 中，假设是拜占庭节点占比 `f < n/3`，且行为任意。而 2026 年的现实是：LLM 节点即便诚实也带有随机性、跨模型之间存在相关性、还会受彼此输出的影响。你不能把它们当作独立的伯努利投票者。

经典 BFT（PBFT，1999）并没有错——只是不完整。它能处理任意的比特翻转，但处理不了"三个诚实的智能体因为共享训练数据而共享同一个幻觉"。本课从 PBFT 的基础出发，逐层叠加三种 2025-2026 年的改进方案。

## 核心概念

### 经典 BFT 能给你什么

实用拜占庭容错（Practical Byzantine Fault Tolerance，Castro & Liskov，OSDI 1999）可容忍 `f < n/3` 个拜占庭节点。协议有三个阶段（pre-prepare、prepare、commit）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意的节点之间，对单一值达成一致。

这些保证很强，但建立在以下假设之上：

1. **故障相互独立。** 拜占庭节点不会串通。
2. **诚实节点是真正诚实的。** 诚实输出的正确性不是问题；协议只负责消除分歧。
3. **问题存在标准答案。** 即便共识落在一个错误事实上，它仍然是共识。

LLM 智能体把这三条全部打破。运行同一基础模型的两个智能体共享同样的缺陷。一个"诚实"的 LLM 照样会产生幻觉。而在模糊问题上，"真相"就是智能体们决定的那个结果——并不存在外部的裁判。

### 三种 LLM 特有的攻击

**拜占庭谎言（Byzantine lie）。** 一个智能体故意输出错误答案。只要 `f < n/3`，经典 BFT 就能应对。

**谄媚从众（Sycophantic conformity）。** 一个智能体在投票前先读别人的答案，然后附和最后一个发言者。它没有恶意，却会与最响亮的声音产生相关性。经典 BFT 防不住这一点，因为这个智能体能通过所有签名校验。

**相关错误单一文化（Correlated-error monoculture）。** 三个智能体共享同一基础模型，它们幻觉出同一个错误答案。多数派是错的。经典 BFT 无能为力，因为三者是"诚实地"达成了一致。

### 2025-2026 年的应对方案

**CP-WBFT**（arXiv:2511.10400）——置信度探针加权 BFT（Confidence-Probed Weighted BFT）。每个投票者在答案上附加一个置信度探针（自报概率，或由独立校准模型给出的预测）。投票权重随置信度缩放。论文报告在完全图上 BFT 性能提升 +85.71%。缓解对象：谄媚从众（从众的智能体对自己主动提出的立场往往置信度很低）。

**DecentLLMs**（arXiv:2507.14928）——无领导者架构。工作智能体并行提出方案，评估智能体为方案打分，最终答案取打分后各立场的几何中位数。在 `f < n/2` 时保持稳健。缓解对象：拜占庭谎言和相关错误（几何中位数对离群点稳健，会被拉向密集簇，而不是带模型偏差的平均值）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT。投票权重由回答质量加上从历史中学习的信任分共同决定。将智能体聚类为核心（Core）和边缘（Edge）；核心智能体必须先达成共识，边缘智能体随后跟进。缓解对象：可扩展性（核心共识规模小、速度快），并部分缓解单一文化问题（核心节点可以按多样性挑选）。

### 实证研究："Can AI Agents Agree?"（arXiv:2603.01213）

该论文在多个前沿模型上测量了标量一致性（LLM 智能体就单一数值达成一致的能力）。发现令人不安：

- 即便没有对手存在，LLM 智能体在标量问题上的分歧率在许多基准上超过 30%。
- 单个采用欺骗性人格的智能体，就能把 Mixture-of-Agents 的共识从诚实基线拉偏 40 个百分点以上。
- 分歧率与模型多样性相关——异构集成比同构集成分歧更多（好处：错误不相关），但收敛也更慢（坏处：达成一致的时间更长）。

结论：BFT 给你的是对齐输出的机制，但它无法告诉你对齐后的输出是否正确。需要结合验证（Phase 16 · 08 角色专业化）、多样性（Phase 16 · 15 辩论变体）和评估智能体（Phase 16 · 24 基准测试）一起使用。

### 精简后的核心协议

面向 LLM 智能体的最小 BFT 回合：

```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

语义聚类这一步是 LLM 特有的变化。"the study reports 4.2%" 和 "4.2% improvement" 这两个答案属于同一个簇。朴素的字符串相等判断会漏掉这一点。生产环境中应使用一个轻量的嵌入模型，或显式的答案规范化。

### 阈值调优

`threshold` 参数决定何时接受、何时重试。太低：你会接受脆弱的多数。太高：你永远什么都接受不了。经验范围：`n=5-7` 个智能体时取 0.5-0.67，`n` 更小时取更高值。低于阈值时，升级交给人工或换一组智能体集成处理。

### 共识帮不上忙的场景

- **模糊问题。** 如果问题没有标准答案，共识只是一种观点。就该如实称之为观点。
- **复合问题。** "写代码并解释它"——这是两个答案。应分别独立投票。
- **对抗性多轮交互。** 如果智能体能观察前几轮并模仿（Du 2023 辩论），它们就会开始不顾真相地互相附和。要限定轮数（通常 2-3 轮）。

## 从零实现

`code/main.py` 实现了：

- `AgentVoter` —— 带有（答案，置信度）的脚本化策略。
- `MajorityVote` —— 经典的简单多数。
- `CPWBFT` —— 带语义聚类的置信度加权投票。
- `DecentLLMs` —— 对打分后的方案做几何中位数聚合。
- `Scenario` —— 在三种攻击模式下分别运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体抄袭它看到的第一个答案，置信度也照抄。
3. `monoculture`：三个智能体以中等置信度共享同一个错误答案（相关错误）。

运行：

```
python3 code/main.py
```

预期输出：一张（攻击，聚合器）-> 最终答案的表格，并高亮正确答案。简单多数在单一文化场景下失败。CPWBFT 的置信度加权能缓解谄媚攻击。当单一文化群体不足总体的一半时，DecentLLMs 的几何中位数会被拉向诚实簇。

## 交付产物

在交付任何共识机制之前：

- **至少用上述三种模式做攻击测试。** 你的协议应当以可预测的方式失败，而不是悄无声息地失败。
- **记录每一个少数簇并保留来源信息。** 少数簇是你发现相关错误的预警系统。
- **强制限定轮数。** 不要"辩论到达成一致为止"——那是在奖励谄媚行为。
- **将一致性与正确性分开。** 共识输出交给验证器；验证器独立于集成本身。
- **监控一致率。** 一致率骤升意味着从众偏差；骤降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认简单多数在单一文化攻击下失败，而当单一文化群体的置信度低于 0.7 时，CPWBFT 能部分缓解该攻击。
2. 添加第四种攻击模式：**沉默弃权**——一个智能体拒绝回答（"I don't know"）。每个聚合器应当如何处理弃权？实现你的选择。
3. 把语义聚类从字符串规范化换成嵌入相似度（使用任意开源嵌入模型）。谄媚攻击会发生什么变化？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探针校准步骤（用一个独立的校准模型检查每个智能体自报的置信度）。测量在单一文化场景下的准确率提升。
5. 阅读 "Can AI Agents Agree?"（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体、一个标量问题、欺骗性人格提示词。CPWBFT 或 DecentLLMs 能识破它吗？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------------|------------------------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 年提出的共识协议，可容忍 `f < n/3` 个任意故障。 |
| 拜占庭（Byzantine） | "任何坏行为" | 一个节点可以撒谎、丢弃消息、悄无声息地失效——除了安全崩溃之外的任何行为。 |
| 置信度探针（Confidence probe） | "你有多确定？" | 附加在选票上的概率值，由智能体自报或校准器预测。 |
| 语义聚类（Semantic clustering） | "同一个答案，不同的措辞" | 在计票之前把等价的答案归为一组。 |
| 几何中位数（Geometric median） | "稳健中心" | 到各样本点距离之和最小的那个点。与均值不同，它对离群点稳健。 |
| 单一文化（Monoculture） | "同一个模型，同样的失败" | 智能体共享训练数据或基础模型时产生的相关错误。 |
| 谄媚从众（Sycophantic conformity） | "附和响亮的声音" | 智能体的投票偏向最先/最响亮的发言者。 |
| 核心/边缘（Core/Edge） | "层次化 BFT" | WBFT 的划分方式：先由小规模核心达成共识，边缘节点跟进。可约束延迟。 |
