# 投票、自洽性与辩论拓扑

> 最廉价的聚合方式：采样 N 个独立智能体，然后多数投票。Wang et al. 2022 的自洽性（self-consistency）方法就是用一个模型采样 N 次实现的。多智能体在此基础上引入**异构**智能体来摆脱单一文化（monoculture）——不同的模型、不同的提示词、不同的温度、不同的上下文。在多数投票之外，辩论拓扑同样重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星型 / 链式 / 树型 / 图型协调结构，发现**图型在研究类任务上表现最佳**，但超过约 4 个智能体后会出现"协调税"。AgentVerse（ICLR 2024）记录了两种涌现模式——自愿行为和从众行为——而从众既是特性（达成共识）也是风险（群体思维，第 24 课）。本课将绘制拓扑空间地图，构建每种变体，并测量协调税。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 07 (Society of Mind and Debate), Phase 16 · 14 (Consensus and BFT)
**Time:** ~75 minutes

## 问题背景

辩论可以提升准确率（Du et al., arXiv:2305.14325），也可能降低准确率。辩论是否有效取决于四个结构性选择：

1. 谁与谁对话（拓扑）。
2. 进行多少轮（Du 2023：轮数和智能体数量各自独立地起作用）。
3. 智能体是否异构（不同的基础模型打破单一文化）。
4. 是否存在对抗性声音（钢人论证 vs. 稻草人论证）。

把"跑 5 个智能体然后投票"直接套到任务上的团队，效果往往不如单个智能体。这些失败并非随机，而是与拓扑和异构性紧密相关。本课就是这张拓扑地图。

## 核心概念

### 自洽性：单模型基线

Wang et al. 2022（"Self-Consistency Improves Chain of Thought Reasoning"）在温度 > 0 的设置下对同一个模型采样 N 次，然后对推理路径的答案进行多数投票。在 GSM8K 上的结果：N=40 个样本相比单次贪心解码有显著提升。自洽性是多智能体投票的单智能体前身。

局限：自洽性只使用一个基础模型。错误在构造上就是相关的。如果模型存在系统性偏差，所有 N 个样本都会共享这个偏差。

### 多智能体投票：异构扩展

把 N 个样本替换为 N 个*不同的*智能体。不同的基础模型（Claude、GPT、Llama）、不同的提示词、不同的工具访问权限。好处：错误不相关。代价：不同智能体的成本各不相同；协调它们会增加开销。

异构辩论在 2026 年的标准名称是 **A-HMAD**——对抗性异构多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。尚未被普遍采用，但论文中用这个术语指代"不同模型之间辩论，从而减少单一文化崩塌带来的相关性错误"。

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星型（Star）：一个中枢，其他所有智能体只与中枢通信。等价于没有反向通道的监督者-工作者结构。
链式（Chain）：线性结构，每个智能体看到前一个的输出。类似流水线。
树型（Tree）：层级结构，用于层级化智能体系统（第 06 课）。
图型（Graph）：任意到任意。包括全连接团（clique）和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包含研究、编码、规划的任务套件上对星型、链式、树型、图型做了基准测试。关键测量结果：

- **图型**拓扑在研究类任务上胜出。信息任意到任意流动；智能体可以互相批评。
- **星型**在快速回答的事实类任务上胜出。中枢负责过滤和整合。
- **链式**在分步流水线（分阶段精炼）上胜出。
- **协调税**在图型拓扑中超过约 4 个智能体后出现。墙钟时间和 token 成本的增长快于质量提升。

4 个智能体的上限是经验性的，不是根本性的。它反映了 2026 年 LLM 的上下文容量：每个智能体的上下文会被同伴的输出填满，一旦所有智能体都能看到彼此，新增第 N+1 个智能体的边际价值就会下降。

### 多智能体辩论策略（"Should we be going MAD?"）

arXiv:2311.17371 是 2023 年关于 MAD 策略的综述。被其他研究复现的关键发现：在相同预算下，*结构上类似于*自洽性（独立采样 + 聚合）的 MAD 变体往往不如自洽性。MAD 在智能体真正异构、且辩论具有对抗结构（有一个智能体专门唱反调）时帮助最大。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了两种即使没有显式设计也会从多智能体辩论中涌现的行为：

- **自愿（Volunteer）。** 智能体主动提出帮助（"我可以接手下一步"）。有用：它把工作分配给最适合某个子任务的智能体。
- **从众（Conformity）。** 智能体调整自己的立场去迎合批评者，即使批评者是错的。这是辩论场景下的谄媚行为（第 14 课）。

从众正是"辩论到达成一致为止"会奖励霸凌者的原因。有界轮数加上独立裁判可以缓解这个问题。

### 异构性：真正影响准确率的旋钮

2024-2026 年实践文献中的一个模式：把你 N 个智能体中的一个换成不同的基础模型，带来的准确率提升比把 N 增加 1 更大。直觉在于单一文化——每个新的独立错误来源，价值高于一个额外的相关样本。

在极限情况下，异构性胜过数量。在大多数有清晰标准答案的任务上，三个不同的模型胜过同一个模型的五个副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了"陪审团"——一小组专门化的智能体，在每个阶段通过投票来精炼答案。与简单多数投票不同，陪审团有角色分工：一个智能体负责交叉质询，一个提供上下文，一个评估可信度。陪审团方法是简单投票（廉价、易陷入单一文化）与完整 MAD（昂贵、易陷入从众）之间的折中。

### 投票加辩论何时占优

- 问题有标准答案（事实、数学、代码行为）。投票收敛才有意义。
- 智能体可以访问不同的信息源或工具（异构性可得）。
- 轮数有界（通常 2-3 轮），且有独立的裁判或验证者。
- 预算允许 3-5 个智能体。在图型拓扑上超过 5-7 个后，协调税占主导。

### 投票加辩论何时有害

- 问题是观点型的。智能体会向看起来最自信的答案收敛，而不是最正确的答案。
- 所有智能体共享同一个基础模型。单一文化让共识失去意义。
- 轮数无界。从众每次都会获胜。
- 任务很简单。N=5 的单智能体自洽性更便宜，准确率不相上下。

## 从零实现

`code/main.py` 实现了：

- `run_star(agents, hub, question)` —— 中枢轮询每个工作者并聚合。
- `run_chain(agents, question)` —— 顺序精炼。
- `run_tree(root, children, question)` —— 层级结构，深度为 2 的聚合。
- `run_graph(agents, question, rounds)` —— 全对全辩论，轮数有界。
- 一个脚本化的异构性旋钮：每个智能体有一个 `error_bias`，表示其系统性错误倾向。
- 一个测量框架，在 N=3、5、7 下运行每种拓扑，报告（accuracy, total_tokens, wallclock_simulated）。

运行：

```
python3 code/main.py
```

预期输出：一张拓扑 × N → (accuracy, tokens, latency) 的表格。在研究类任务上，图型在 N=3-5 时胜出；星型在快速事实类任务上胜出；图型在 N=7 时显现协调税（延迟膨胀的速度快于准确率提升）。

## 生产实践

`outputs/skill-topology-picker.md` 是一个技能：它读取任务描述，推荐一种拓扑（星型 / 链式 / 树型 / 图型）、一个 N（智能体数量）、一个异构性配置（使用哪些基础模型）以及一个轮数上限。

## 交付产物

对于任何集成系统（ensemble）：

- 从使用一个强基础模型的 **N=5 自洽性**开始。这是廉价基线。
- 如果准确率重要，升级到 **N=3 的异构投票**。测量提升幅度。
- 只有当任务有结构（研究类、多步骤）且轮数可以有界时，才升级到**辩论拓扑**。
- 始终记录少数派簇。当少数派持续正确时，你得到了一个多样性信号。
- 在准确率之外同时基准测试墙钟时间和 token。"准确率更高但成本 10 倍"是一个商业决策。

## 练习

1. 运行 `code/main.py`。绘制图型拓扑的协调税曲线：accuracy vs N、tokens vs N。曲线在哪个 N 处出现拐点？
2. 实现 A-HMAD：三个智能体，刻意设置不同的偏差。在第 14 课的单一文化攻击上，全部同偏差的基线与 A-HMAD 相比如何？
3. 给图型拓扑加一个"裁判"角色：不投票，只给最终共识打分。这是否改变了涌现的从众行为？
4. 阅读 AgentVerse 论文（ICLR 2024）。识别你的实现中表现最强的涌现行为是哪一种。能否通过修改提示词诱发出相反的行为？
5. 阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。用你的测量框架在论文中的一个任务上复现"图型在研究类任务上胜出"的结果。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 自洽性（Self-consistency） | "采样 N 次，投票" | Wang 2022。单模型，N 个温度>0 的样本，对推理路径做多数投票。 |
| 异构性（Heterogeneity） | "不同的模型" | 由不同基础模型或提示词族组成的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在多轮中交换批评意见的统称。见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | 强调不同模型 + 对抗结构的 MAD 变体。 |
| 拓扑（Topology） | "谁与谁对话" | 星型、链式、树型、图型。决定信息流动方式。 |
| 协调税（Coordination tax） | "收益递减" | 图型拓扑超过约 4 个智能体后，成本增长快于质量提升。 |
| 自愿行为（Volunteer behavior） | "主动帮忙" | AgentVerse 涌现模式：智能体主动提出承担一个步骤。 |
| 从众行为（Conformity behavior） | "压力下的附和" | AgentVerse 涌现模式：智能体向批评者靠拢。 |
| 陪审团（Jury） | "小型专家小组" | Sibyl 式集成，带角色分工（质询者、上下文提供者、评分者）。 |
