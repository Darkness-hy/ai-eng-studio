# 协商与议价

> 智能体之间会就资源、价格、任务分配和条款进行协商。2026 年的基准测试结果非常明确：NegotiationArena（arXiv:2402.05863）表明 LLM 可以通过人设操纵（"表现绝望"）将收益提升约 20%；"Measuring Bargaining Abilities"（arXiv:2402.15813）表明买方比卖方更难当，而且模型规模帮不上忙——他们提出的 **OG-Narrator**（确定性报价生成器 + LLM 叙述器）将成交率从 26.67% 推高到 88.88%；大规模自主协商竞赛（arXiv:2503.06416）运行了约 18 万次协商，发现**隐藏思维链**的智能体通过对对手隐藏推理过程而获胜；Bhattacharya 等人 2025 年基于哈佛谈判项目（Harvard Negotiation Project）指标的研究将 Llama-3 评为最高效、Claude-3 最激进、GPT-4 最公平。本课将实现合同网协议（Contract Net Protocol，FIPA 的前身，见第 02 课），接入 LLM 风格的买方/卖方，运行 OG-Narrator 式的分解方案，并测量每一个结构性选择如何改变成交率。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 02 (FIPA-ACL Heritage), Phase 16 · 09 (Parallel Swarm Networks)
**Time:** ~75 minutes

## 问题背景

两个智能体需要就一个价格达成一致。如果只靠纯语言提示词放任自流，2024-2026 年的 LLM 成交率低得惊人（在 arXiv:2402.15813 的严格参数化议价场景中约为 27%）。扩大模型规模解决不了问题：GPT-4 在议价的*结构*上并不比 GPT-3.5 更强，它只是更擅长议价的*语言*。

根本问题在于 LLM 把两件事混在了一起——决定报价和表述报价。OG-Narrator 把它们分开：由确定性报价生成器计算数值动作，LLM 只负责叙述。成交率随之跃升到约 89%。

这印证了多智能体领域的一个经典结论：把机制与通信层解耦才是赢家。合同网协议（FIPA, 1996; Smith, 1980）是任务市场机制的标杆。把一个 LLM 插到叙述槽位上，你就得到了一个现代的 LLM 驱动任务市场。

## 核心概念

### 一段话讲清合同网

Smith 1980 年的合同网协议：一个**管理者（manager）**广播一条**提案征集（call for proposals, cfp）**；**投标者（bidders）**用包含各自报价的 **propose** 消息回应；管理者选出一个赢家，向赢家发送 **accept-proposal**，向落选者发送 **reject-proposal**。赢家执行任务。可选消息：**refuse**（投标者拒绝提案）。FIPA 将其规范化为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 能赢

"Measuring Bargaining Abilities of Language Models"（arXiv:2402.15813）观察到：

- LLM 经常违反议价规则（在荒谬的价格上报价、无视对方的 ZOPA）。
- 它们的锚定能力很差（接受糟糕的首次报价；还价金额带有象征意味而非策略性）。
- 单靠规模无法解决这些问题。更大的模型只是把语言写得更像样，策略错误依旧。

OG-Narrator 的分解方案：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ offer generator  │ price → │  LLM narrator    │ → message
           │  (deterministic) │        │  (writes the     │
           │                  │        │   human-style    │
           └──────────────────┘        │   accompaniment) │
                                       └──────────────────┘
```

报价生成器是一个经典的协商策略：Rubinstein 议价模型、Zeuthen 策略，或者一个简单的价格层面以牙还牙（tit-for-tat）策略。LLM 负责叙述。消息中包含确定性的价格和自然语言的措辞包装。

成交率跃升的原因在于：

- 价格始终落在议价区间内。
- 锚点是策略性的，而非情绪化的。
- LLM 做的是它擅长的事：写作。

### NegotiationArena 的发现

arXiv:2402.05863 提供了公认的基准。核心发现：

- LLM 可以通过采用人设（"我急着在周五前把它卖掉"）将收益提升约 20%——人设操纵是真实有效的战术。
- 公平/合作型智能体会被对抗型智能体利用；防御需要明确的反向姿态。
- 在大约 40% 的基准场景中，对称配对会收敛到不公平的结果。

这并不是说"LLM 是糟糕的谈判者"，而是说"LLM 谈判起来太像人类了，包括那些可被利用的部分"。

### 思维链隐藏

大规模自主协商竞赛（arXiv:2503.06416）在多种 LLM 策略之间运行了约 18 万次协商。赢家都对对手隐藏了自己的推理过程：

- 如果一个智能体在公开可见的草稿区里写下"我最多只出 $75；我的保留价是 $70"，对手会读到它。
- 赢家在私有空间计算策略；输出通道里只包含报价和最低限度的必要叙述。

这是经典博弈论（Aumann 1976 关于理性与信息的研究）在 2026 年的回响：暴露你的私有估值会损失收益。LLM 对此毫无直觉，会心安理得地把自己的保留价写进推理痕迹里，而这些痕迹对对手是可见的。

工程结论：把私有草稿区上下文与公开消息上下文分开。这不是可选项。

### Bhattacharya 等人 2025——模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 遵循度、利益互惠）上：

- **Llama-3** 达成交易的效率最高（成交率 + 收益）。
- **Claude-3** 是最激进的谈判者（锚点高、让步晚）。
- **GPT-4** 最公平（不同配对之间的收益方差最小）。

这是 2025 年的一份快照。重点不在于 2026 年 4 月哪个模型会赢——而在于不同的基座模型有着持久稳定的谈判风格。异构集成（第 15 课）将其纳入为一种多样性来源。

### 基于合同网 + LLM 的任务分配

合同网在 LLM 多智能体场景下的现代复用方式：

1. 管理者智能体把任务分解成若干单元。
2. 向工作者智能体广播带任务描述的 `cfp`。
3. 每个工作者返回一份报价：`(price, eta, confidence)`，其中 price 可以是 token、计算单元或美元。
4. 管理者选出赢家（根据任务可以是单个或多个）并授标。
5. 落选的工作者可以自由竞标其他任务。

这种模式能轻松扩展到 100 个以上的工作者，因为协调方式是广播-应答，而不是同步聊天。生产环境中的应用：Microsoft Agent Framework 的编排模式、一些 LangGraph 实现。

### LLM-Stakeholders 交互式协商

NeurIPS 2024（https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf）引入了带**秘密分数**和**最低接受阈值**的多方可计分博弈。每个利益相关者拥有私有效用；LLM 必须从消息中推断它们。这是把双方议价推广到 N 方联盟形成的泛化形式。对于工作者能力各异的生产级任务市场而言，这项工作很有参考价值。

### 叙述与机制分离法则

纵观 2024-2026 年的所有协商基准，始终成立的工程法则是：

> 让 LLM 负责叙述。不要让 LLM 计算报价。

如果报价必须是一个数字（价格、ETA、数量），就根据协商状态确定性地生成它，由 LLM 负责措辞包装。如果报价是一个提案结构（任务分解、角色分配），可以让 LLM 起草，但在发送前必须用 schema 验证并做约束检查。

## 从零实现

`code/main.py` 实现了：

- `ContractNetManager`、`ContractNetTask`、`Bid`——管理者 + 投标者，广播 cfp、收集提案、授标。
- `og_narrator_bargain(state, rng)`——OG-Narrator 买方：以 Zeuthen 风格向中点确定性让步。
- `seller_response(state, rng)`——确定性的卖方还价策略（两种风格共用的结构性基准真值）。
- `naive_llm_bargain(state, rng)`——模拟一个完全由 LLM 驱动的议价者：选价方差大，经常落在 ZOPA 之外。
- 测量：在 1000 次试验上统计成交率，每次试验都重新采样保留价。

运行：

```
python3 code/main.py
```

预期输出：朴素 LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；这 15-25 个百分点的差距就是把报价生成与叙述解耦带来的结构性优势。此外还有一个三个投标者、一个任务的合同网任务市场分配示例。

## 生产实践

`outputs/skill-bargainer-designer.md` 设计了一套议价协议：谁生成报价（确定性还是 LLM）、谁负责叙述、私有草稿区如何与公开消息隔离，以及如何监控成交率。

## 交付产物

生产级议价检查清单：

- **隔离草稿区。** 私有状态绝不进入对手的上下文。这一条没有商量余地。
- **确定性报价生成。** 价格、数量、ETA：用计算得出，不要用提示词。
- **校验所有传入的报价**，使其符合 schema。在协议边界处直接拒绝 ZOPA 之外的报价。
- **限定轮数。** 最多 3-5 轮；陷入僵局时升级给调解者。
- **持续测量成交率和收益方差。** 成交率下降是一种症状——通常意味着提示词漂移或对手侧的攻击。
- **记录所有被拒绝的提案**，附上确定性的理由。对合同网管理者来说，落选的投标者需要理解自己为什么落选。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上胜过朴素 LLM。胜出多少？
2. 实现**基于人设的收益提升**（arXiv:2402.05863）——买方仅在叙述中采用"急着这周买到"的人设，报价生成器保持不变。成交率或收益会变化吗？
3. 实现思维链**隐藏**：维护一个不传给对手的私有草稿区字符串。如果你不小心泄露了它（通过交换通道来模拟），会发生什么？
4. 把合同网扩展为带保留价的 N 投标者拍卖。当所有出价都超过保留价时，管理者如何在最低价和最高质量之间取舍？你会选择哪种授标规则，为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的研究。实现两个风格不同的议价者（激进 vs 公平）。在对称和非对称配对下测量收益方差。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------------|------------------------|
| Contract Net | "任务市场" | Smith 1980, FIPA 1996。cfp + propose + accept/reject。任务市场的经典范式。 |
| ZOPA | "可能达成协议的区间" | 买方最高价与卖方最低价之间的重叠区间。落在区间外的报价不可能成交。 |
| BATNA | "谈判协议的最佳替代方案" | 这笔交易失败时你的退路。它决定了你的保留价。 |
| OG-Narrator | "报价生成器 + 叙述器" | 分解方案：确定性报价，LLM 叙述。 |
| Zeuthen 策略 | "风险最小化让步" | 经典报价生成器，基于风险限度做出让步。 |
| Rubinstein 议价 | "交替报价均衡" | 带折扣因子的无限期议价的博弈论模型。 |
| CoT 隐藏 | "藏好你的推理" | arXiv:2503.06416 中的赢家保有私有草稿区；公开通道只展示报价。 |
| 人设操纵 | "情绪化表演" | arXiv:2402.05863：靠绝望/紧迫人设获得约 20% 的收益提升。 |
