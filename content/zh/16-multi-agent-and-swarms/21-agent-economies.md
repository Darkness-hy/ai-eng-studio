# 智能体经济、代币激励与声誉

> 长时程自主智能体（METR 给出的 1 小时到 8 小时工作曲线）需要经济上的行动能力。正在形成的**五层技术栈**是：**DePIN**（物理算力）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产级的智能体激励网络包括 **Bittensor**（TAO 子网奖励面向特定任务的模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币）以及 **Gonka**（基于 Transformer 的 PoW，把算力重新分配给有生产价值的 AI 任务）。学术方面：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值贡献归因（Shapley-value credit attribution）** 来公平地奖励有贡献的智能体；Google Research 的 "Mechanism design for large language models" 提出在单调聚合条件下采用次价支付的**代币拍卖**。本课会构建一个最小化的智能体市场，把 Shapley 值贡献归因应用到多智能体流水线上，并运行一次次价代币拍卖，让这套博弈论机制真正落到实处。

**Type:** Learn

## 问题背景

当智能体共同创造价值却需要被单独奖励时，多智能体系统就会变得棘手。经典机制——平均分配、最后贡献者全拿——要么不公平，要么容易被钻空子。基于联盟的 Shapley 值奖励在构造上就是公平的，但计算代价高昂。2025-2026 年的文献给出了实用的近似方法：Shapley 采样、单调聚合拍卖，以及由已确认贡献累积而成的链上声誉。

在贡献归因之外，这个领域已经转向真正的经济型智能体：Bittensor TAO 奖励为子网专属模型微调提供算力的挖矿行为，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 的使用，Gonka 把 Transformer 工作量证明重新导向有生产价值的 AI 任务。能自主交易的智能体今天已经存在；问题在于如何对齐激励。

本课把智能体经济当作一个具体的问题族来处理——贡献归因、机制设计和声誉——并用最少的数学逐一实现，让这些想法真正扎根。

## 核心概念

### 智能体经济的五层技术栈

1. **DePIN（物理算力）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。它并非智能体专属；智能体只是使用它。
2. **身份。** W3C 去中心化标识符（Decentralized Identifiers, DID）给每个智能体一个不依赖任何平台的持久 ID。声誉累积在 DID 上。Agent Network Protocol（ANP）把 DID 用作发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他各阶段所构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体可以用自己的余额支付 gas，而无需持有 ETH。智能体可以为服务、为彼此或为算力付费。
5. **治理。** 智能体 DAO：人类*和*智能体共同对协议变更投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都用满五层。Bittensor 用了第 1、2 层，部分用了第 3 层，部分用了第 4 层，完全没用第 5 层。OpenAI 的智能体除了第 3 层之外什么都不用。这个技术栈是一张参考地图，不是硬性要求。

### Bittensor、Fetch.ai、Gonka——实际在跑什么

**Bittensor（TAO）。** 子网对应专门化的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其排名；按质押加权的评分分配 TAO 奖励。每个子网有自己的评测方式。其经济学启示是：为特定任务的输出质量付费，而不是为消耗的算力付费。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLM 运行在 Fetch.ai 的网络上；用户用 FET 代币支付推理费用。"智能体即对等节点"的叙事在这里更突出：Fetch 上的一个智能体可以调用另一个智能体完成任务并用 FET 付款。

**Gonka。** Transformer 工作量证明："工作"就是 Transformer 的前向传播。矿工通过运行带有已知正确输出（来自训练数据）的推理任务赚取收益。这是资源有产出的 PoW，而非基于哈希的 PoW。

截至 2026 年 4 月，这三者都已达到生产级。收益分配方式各不相同：Bittensor 按子网验证者评判的相对质量发放奖励；Fetch 按付费用户衡量的效用发放奖励；Gonka 按可验证的推理工作量发放奖励。

### Shapley 值贡献归因

三个智能体协作完成一项任务，输出得分 0.8。各自贡献了多少？

Shapley 值：满足四条公理（有效性、对称性、线性性、零贡献者）的唯一信用分配。对智能体 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中位于 `i` 之前的智能体集合。实际操作就是：枚举所有排列，记录每个智能体在每个排列中的边际贡献，然后取平均。

N=3 个智能体时有 6 个排列。N=10 时有 360 万个——所以实践中要对排序进行采样，而不是穷举。

### 用于聚合的次价拍卖

Google Research（"Mechanism design for large language models"）提出用次价代币拍卖来聚合 LLM 输出。设定：N 个智能体各自提出一个补全结果；每个智能体对自己被选中有一个私有估值。拍卖方选出估值最高的提案，但只收取*第二高*的估值。在单调聚合条件下（价值只取决于哪个提案被选中，而不取决于有多少人出价），这一机制是诚实的——智能体会按真实估值出价。

这对 LLM 系统的意义在于：你可以把补全任务外包给多个定价不同的智能体；拍卖会挑出最优方案并公平付费，而智能体没有任何谎报的动机。

### 声誉资本

绑定到 DID 的声誉分数由已确认的贡献累积而成。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

其中衰减因子 `alpha` 接近 1。声誉的特点：

- 读取成本低，便于路由决策（"把难任务派给高声誉智能体"）。
- 伪造成本高（需要长期累积，且绑定到 DID）。
- 可以被罚没：未通过验证的贡献会扣分。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）组合了三件事：DID 身份、Shapley 值贡献归因，以及一个简单的拍卖机制。其核心主张是：把贡献归因这一步去中心化，可以让系统可审计，并免疫单点操纵。

### 经济机制会在哪里崩坏

- **价格预言机操纵。** 只要信用函数能被钻空子，智能体就一定会去钻。每个机制都需要对抗性测试。
- **女巫攻击（Sybil attack）。** 一个运营者拉起 N 个假智能体来夸大自己的贡献。DID 能减缓但无法阻止这种攻击；声誉的伪造成本才是缓解手段。
- **验证成本。** 贡献归因的公平程度取决于验证者。如果验证很便宜（小型 LLM），它就可能被钻空子；如果很昂贵（人工评审团），系统就无法扩展。
- **监管悬置。** 智能体经济与金融监管交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法辖区都处于法律灰色地带。

### 智能体经济何时有意义

- **运营者异构的开放网络。** 没有任何一个团队控制所有智能体。
- **输出可验证。** 没有验证，贡献归因就只是猜测。
- **长时程工作流。** 一次性任务无法从声誉累积中获益。
- **代币化支付在你所在的司法辖区合法可行。**

在封闭的企业系统中，经济机制会让位于更简单的分配方式（管理者派活，指标走内部流程）。这套经济学文献主要适用于开放网络。

- `Reputation` — 绑定 DID 的声誉，带指数衰减和罚没。
- Demo 1：三个智能体协作，精确 Shapley 完成贡献归因。
- Demo 2：五个智能体竞标一个任务名额；次价拍卖选出赢家并确定付款。
- Demo 3：向声誉各异的智能体做 100 轮任务分配；按声誉加权的路由胜过随机路由。

```
python3 code/main.py
```

预期输出：每个智能体的 Shapley 值；展示真实出价均衡的拍卖结果；声誉加权路由在预热后比随机路由有 10-20% 的质量提升。

## 交付产物

在 2026 年运营一个智能体经济：

- **从声誉开始，而不是代币。** 声誉实现成本低，单独使用也有价值；代币会带来法律和经济上的复杂性。
- **先验证，再奖励。** 没有独立验证步骤就绝不分配信用。自报质量会滋生女巫博弈。
- **用 Shapley 采样，不要 Shapley 精确计算。** 采样 100-1000 个排序；精确枚举无法扩展。
- **给衰减因子设上限，给声誉设下限。** 无界的衰减会抹掉合法贡献者；衰减太慢则会奖励吃老本的高声誉智能体。
- **对机制做对抗性审计。** 在开放网络之前先跑红队场景。每个机制背后都有一套博弈论；你要先于攻击者找到漏洞。

## 练习

1. 运行 `code/main.py`。确认 Shapley 值之和等于总价值（有效性公理）。改变价值函数；Shapley 分配是否朝预期方向变化？
2. 实现 Shapley *采样*（对 K 个排序做 Monte Carlo）。K 如何影响近似精度？在 N=4 时与精确值对比。
3. 在拍卖前实现一个联盟形成步骤：智能体可以合并成团队并作为整体出价。哪些联盟会形成？结果是否比单独出价更 Pareto 优？
4. 阅读 Google Research 的机制设计博文。找出一条一旦被违反就会破坏诚实性的假设。在 LLM 场景下，这种失效模式长什么样？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在一个合成任务上对 10 个智能体实现他们的 Shapley 步骤。精确计算需要多久？采样 100 次能逼近到什么程度？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| DePIN | "去中心化物理基础设施" | 代币激励的算力/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | 可移植 ID 的 W3C 规范。智能体声誉绑定到 DID，而非某个平台。 |
| ERC-4337 | "账户抽象" | 可以代付 gas 的合约账户，使智能体支付成为可能。 |
| Shapley 值 | "公平的贡献归因" | 满足有效性、对称性、线性性、零贡献者的唯一分配。 |
| 次价拍卖 | "Vickrey 拍卖" | 诚实机制：赢家支付第二高出价。与单调聚合兼容。 |
| 声誉资本 | "累积的质量分数" | 由已确认贡献形成的 DID 绑定分数；随时间衰减。 |
| 智能体 DAO | "智能体 + 人类共同治理" | 智能体投票者享有一等地位的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 额度 | "代币计价单位" | Bittensor TAO、Fetch.ai FET、各类 DePIN 代币。 |
