# 心智社会与多智能体辩论

> Minsky 在 1986 年提出的核心命题——智能是由各类专家组成的社会——每过十年就会被重新发现一次。2023 年，Du 等人把它变成了一个具体的算法：多个 LLM 实例各自提出答案，阅读彼此的答案，互相批评并更新。经过 N 轮之后，它们收敛到一个共识，在六项推理与事实性任务上击败了零样本 CoT 和反思（reflection）。有两个发现尤为重要：**多个智能体**和**多轮交互**各自独立地贡献了提升。"社会"胜过单智能体的独白；多轮交流胜过一次性投票。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 04 (Primitive Model)
**Time:** ~60 minutes

## 问题背景

自洽性（self-consistency）——对同一个模型多次采样并取多数答案——是成本最低的推理增强手段。它确实有效，但很快就会饱和：把采样数翻倍，也看不到再一次有意义的提升。

辩论打破了这种饱和。不再是从一个模型抽取 N 个独立样本，而是让 N 个智能体阅读彼此的推理并修正自己的答案。样本之间的相关性下降（它们不再是独立同分布的），而收敛点往往是正确的——而独立同分布投票却会自信地给出错误答案。

## 核心概念

### Du 等人 2023 年的算法

出自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体各自对问题给出初始答案。
2. 对于第 r = 2..R 轮：每个智能体看到其他智能体在第 r-1 轮的答案，并被要求"参考这些答案，给出你更新后的答案"。
3. R 轮结束后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、人物传记、MATH 以及事实性基准上做了测试。辩论稳定地优于 CoT 和 Self-Reflection。

### 两个独立的调节旋钮

来自同一篇论文的消融实验：

- **只增加智能体数量**（1 轮，N 个智能体多数投票）在多数任务上胜过单智能体，但会进入平台期。
- **只增加轮数**（1 个智能体反复看自己之前的推理）几乎没有帮助——这正是反思方法已知的弱点。
- **两者结合**才产生大幅跃升。多个智能体之间的多轮交流才是收益的来源。

### 为什么有效

两种机制：

1. **暴露于分歧之中。** 当一个智能体看到另一个智能体得出不同结论的推理链时，它必须要么为自己辩护，要么更新答案。无论哪种情况，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **降低相关性误差。** 在自洽性方法中，所有样本都来自同一个模型，因此误差是相关的——平均下来反而得到一个自信的错误答案。不同模型或不同随机种子可以去相关。不同的*经过辩论的观点*能进一步去相关。

### 异构辩论

A-HMAD 及相关后续工作让不同智能体使用*不同的基础模型*。Llama + Claude + GPT 一起辩论可以缓解单一文化坍缩（第 26 课），因为一个模型家族的相关性误差不会被其他家族共享。

缺点：一个弱模型参与辩论时，可能把共识拖向它的错误答案（见 "Should we be going MAD?"，arXiv:2311.17371）。

### NLSOM——扩展到 129 个智能体

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind"，arXiv:2305.17066）把这一思想扩展到 129 个成员的"社会"。结果是：专业化分工与自组织随着规模涌现，系统在视觉问答等任务上优于单智能体。

### 失败模式

- **谄媚级联（sycophancy cascade）。** 所有智能体都顺从听起来最自信的那个智能体，辩论坍缩成最大声者的独角戏。在提示词中设置对抗性角色（"必须有一个智能体论证相反立场"）有帮助。
- **话题漂移。** 多轮辩论会逐渐偏离原始问题。缓解方法：每一轮都重新注入原始问题。
- **计算量爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，且每次调用的上下文都在增长。一个 5 智能体、5 轮的辩论就是 25 次上下文不断膨胀的调用。单个问题的成本可能超过单次 CoT 调用的 10 倍。

## 从零实现

`code/main.py` 在一道数学题上运行一个 3 智能体 × 3 轮的辩论，每个智能体以不同的（可能错误的）初始答案开始。智能体的行为是脚本化的——每个智能体的"更新"方式是按脚本设定的置信度对邻居的答案做加权平均。逐轮日志中可以清楚看到收敛过程。

这个演示展示了两个关键效应：

- 仅一轮交流就能让各智能体向正确答案靠拢。
- 第 2 轮之后的额外轮次收益递减（与 Du 等人观察到的平台期一致）。

运行：

```
python3 code/main.py
```

## 生产实践

`outputs/skill-debate-configurator.md` 为新任务配置一场辩论：智能体数量、轮数、异构性（同一模型还是混合模型）、角色分配（对称还是设置一个对抗角色）。它还会在运行前估算 token 成本。

## 交付产物

如果要把辩论系统投入生产：

- **轮数上限设为 3。** Du 等人表明 3 轮已捕获大部分收益。再多只增加成本，不增加质量。
- **智能体上限设为 5。** 超过 5 个之后，上下文膨胀和成本将占据主导。
- **默认异构。** 智能体池中至少包含两种不同的基础模型。
- **设置对抗席位。** 让一个智能体被提示无论如何都要提出异议。这能打破谄媚级联。
- **记录每一轮日志。** 隐藏中间轮次的辩论系统既无法调试，也无法审计。

## 练习

1. 运行 `code/main.py`，然后把轮数设为 5，观察收益递减。从第几轮开始不再有额外收敛？
2. 添加第四个具有对抗角色的智能体：总是反对当前多数答案。这会破坏还是改善收敛？
3. 绘制（打印）每一轮的一致性分数（持多数答案的智能体占比）。它何时达到 1.0？达到 1.0 是否等价于"正确"？
4. 阅读 Du 等人论文第 4 节的消融实验。用本课代码复现"只加智能体" vs "只加轮数" vs "两者结合"的结果。
5. 阅读 "Should we be going MAD?"（arXiv:2311.17371），列出轮流发言（round-robin）之外的两种辩论变体——例如裁判主导式、辩论链式、对抗式。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|----------------|------------------------|
| 心智社会（Society of Mind） | "Minsky 的想法" | 智能即相互交互的专家集合；1986 年的理论框架如今通过 LLM 辩论得以落地。 |
| 多智能体辩论 | "智能体互相争论" | N 个智能体提出答案，互相批评，经 R 轮修正后多数投票。 |
| 共识 | "它们达成一致了" | 不是认识论意义上的真理——只是持多数答案的占比。可能是自信的错误。 |
| 轮次 | "交流步骤" | 一轮 = 每个智能体阅读其他智能体的答案并更新一次。 |
| 异构辩论 | "混合模型家族" | 使用不同的基础模型来降低误差相关性。 |
| 谄媚级联 | "大家都附和最大声的那个" | 一种辩论失败模式：智能体不管对错都顺从最自信的智能体。 |
| NLSOM | "129 个智能体的社会" | 基于自然语言的心智社会；Zhuge 等人的规模化版本。 |
| 相关性误差 | "同一个模型，同一个 bug" | 自洽性方法饱和的原因；跨不同观点的辩论可以去相关。 |
