# LLM 的差分隐私

> DP-SGD 仍是标准方法——通过噪声注入的梯度更新提供形式化的 (epsilon, delta) 保证。其在计算、内存和效用上的开销很大；参数高效的 DP 微调（LoRA + DP-SGD）是 2025 年的常见配置（ACM 2025）。两类证据存在张力：基于金丝雀（canary）的成员推断攻击（Duan et al., 2024）报告对语言模型的攻击成效有限；而训练数据提取（Carlini et al., 2021; Nasr et al., 2025）却能恢复大量逐字记忆的内容。化解之道（arXiv:2503.06808，2025 年 3 月）：差距在于测量对象不同——插入的金丝雀 vs「最易被提取」的数据。新的金丝雀设计使得无需影子模型即可进行基于损失的 MIA，并首次在真实数据、现实 DP 保证下对 LLM 完成了非平凡的 DP 审计。替代方案：PMixED（arXiv:2403.15638）——通过对下一个 token 分布做专家混合，在推理时进行私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：基于 LLM 反馈的差分隐私逆转（Differential Privacy Reversal via LLM Feedback）——置信度分数泄漏。

- 描述基于 LLM 反馈的差分隐私逆转攻击。

## 问题背景

LLM 会记忆训练数据。Carlini et al. 2021 表明，生产级语言模型可以按需逐字复现训练文本。DP 是形式化的防御手段：让训练过程的输出可证明地对任意单个训练样本不敏感。2024-2025 年的证据表明，DP-SGD 是必要的，但实际部署的 ε 值可能与威胁模型不匹配。

## 核心概念

### (ε, δ)-差分隐私

一个随机化算法 M 满足 (ε, δ)-DP，当且仅当对任意两个仅相差一个样本的数据集和任意事件 S：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ。

含义：输出分布足够接近（由 ε 参数化），因此除以 δ 的概率外，任何单个个体的贡献都无法被可靠推断。

### DP-SGD

Abadi et al. 2016。标准流程：
1. 采样一个 mini-batch。
2. 计算逐样本梯度。
3. 将每个逐样本梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和，并加入标准差为 σ * C 的高斯噪声。
5. 用带噪的梯度和更新参数。

隐私开销由会计机制（accountant）追踪（Moments Accountant、Rényi DP accountant）。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而差异很大；不存在普遍「安全」的默认 ε。已发表的例子在部分 LLM 训练场景中大致落在 ε ≈ 1–10 区间，但这些只是示例——并非推荐默认值。更低的 ε 通常需要更大的噪声，并可能加剧效用损失。

### LoRA + DP-SGD

对前沿模型做完整的 DP-SGD 成本高得难以承受。LoRA（Hu et al. 2022）将梯度更新限制在一个小的适配器上，从而减少逐样本梯度的存储。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证作用于适配器；基础模型保持冻结。

### 2024-2025 年的证据张力

两条证据线：

- **金丝雀 MIA（Duan et al. 2024）。** 在训练数据中插入唯一的金丝雀样本，测量成员推断攻击者能否识别它们。报告称对语言模型成效有限。这暗示 MIA 很难。
- **训练数据提取（Carlini 2021, Nasr et al. 2025）。** 用一个前缀提示模型，测量它能否恢复训练数据中的逐字文本。报告称记忆现象显著。这暗示在与隐私相关的意义上 MIA 是容易的。

2025 年 3 月的化解（arXiv:2503.06808）：两者测量的是不同的东西。MIA 在插入的金丝雀上问「样本 e 是否在 D 中？」提取则问「我能从 D 中恢复什么？」对隐私而言真正重要的是「最易被提取」的样本；金丝雀低估了这一点，因为它们并没有被优化得易于提取。

新的金丝雀设计。无需影子模型的基于损失的 MIA。首次在真实数据、现实 DP 保证下对 LLM 完成非平凡的 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时的私有预测。对下一个 token 分布做专家混合；每个专家只看到训练数据的一个分片；聚合时加噪以实现 DP。完全避开了 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 用 DP-SGD 做 LoRA 微调，从中采样合成数据，再在合成数据上训练下游分类器。

两者都绕开了完整 DP 训练的效用代价，但代价是面对的威胁模型不同。

### 基于 LLM 反馈的差分隐私逆转

2025 年的新兴攻击。利用 DP 训练模型的置信度分数作为预言机（oracle）来重识别个体。即使输出本身不泄漏，置信度分布也可能泄漏。

防御方式：不暴露置信度，或在暴露前对其截断/量化。这是 (ε, δ)-DP 训练之外的额外要求。

## 生产实践

`code/main.py` 在一个玩具二分类数据集上模拟 DP-SGD。你可以扫描噪声乘子 σ 和裁剪范数 C，并追踪 (ε, δ) 预算与准确率代价。「金丝雀攻击」插入一个唯一的训练样本，并测量在 DP 前后对数损失检验能否检测到它。

## 交付产物

本课产出 `outputs/skill-dp-audit.md`。给定某个语言模型部署的 DP 声明，它审计：(ε, δ) 取值、所用的会计机制、MIA 评估协议，以及是否评估过置信度暴露这一攻击向量。

## 练习

1. 运行 `code/main.py`。在 {0.5, 1.0, 2.0} 中扫描 σ，报告 (ε, δ)-准确率权衡。找出效用崩溃的临界点。

2. 实现金丝雀插入和对数损失检验。测量在 σ = 1.0 的 DP-SGD 前后的检测率。

3. 阅读 Nasr et al. 2025 关于训练数据提取的工作。为什么在中等 ε 下提取成功率不会崩塌？这对「以 MIA 作为评估手段」意味着什么？

4. 基于 PMixED（arXiv:2403.15638）设计一个完全在推理时运行的部署方案。PMixED 应对的、而 DP-SGD 应对不了的威胁模型是什么？

5. 勾勒基于 LLM 反馈的 DP 逆转攻击。设计一个限制置信度分数泄漏的对策，并估算其部署成本。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|-----------------|------------------------|
| DP | 「(ε, δ)-差分隐私」 | 形式化隐私：在相邻数据集变化下输出分布保持接近 |
| DP-SGD | 「噪声注入的 SGD」 | 梯度裁剪 + 高斯噪声叠加；标准 DP 训练方法 |
| LoRA + DP-SGD | 「高效私有微调」 | 在低秩适配器上做 DP-SGD；2025 年的标准配置 |
| MIA | 「成员推断」 | 判定某个样本是否在训练数据中的攻击 |
| 金丝雀（Canary） | 「插入的水印样本」 | 用于测量 DP 泄漏的唯一训练样本 |
| PMixED | 「私有推理混合」 | 通过对下一个 token 分布做专家混合实现推理时 DP |
| DP 逆转 | 「置信度泄漏攻击」 | 利用模型置信度作为重识别预言机的攻击 |
