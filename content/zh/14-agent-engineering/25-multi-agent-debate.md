# 多智能体辩论与协作

> Du 等人（ICML 2024，"Society of Minds"）让 N 个模型实例各自独立提出答案，然后在 R 轮中相互批评、迭代修正，最终收敛到一致答案。该方法提升了事实性、规则遵循能力和推理能力。在 token 成本上，稀疏拓扑优于全连接（full mesh）。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 14 · 12 (Workflow Patterns), Phase 14 · 05 (Self-Refine and CRITIC)
**Time:** ~60 minutes

## 学习目标

- 解释辩论协议：N 个提议者，R 轮辩论，最终收敛到共享答案。
- 说明辩论为什么能提升事实性、规则遵循能力和推理能力。
- 解释稀疏拓扑：并非每个辩论者都需要看到所有其他辩论者。
- 用标准库在脚本化 LLM 上实现一个辩论系统，包含全连接和稀疏两种变体；测量 token 成本与准确率的关系。

## 问题背景

Self-Refine（第 05 课）是一个模型批评自己——有「群体思维」的风险。CRITIC（第 05 课）依靠外部工具为批评提供依据——但工具并非总是可用。辩论引入了第三种模式：多个实例、交叉批评、通过分歧达成收敛。

## 核心概念

### Society of Minds（Du 等人，ICML 2024）

- N 个模型实例针对同一个问题独立提出答案。
- 在 R 轮中，每个模型阅读其他模型的提案并提出批评。
- 各模型根据收到的批评更新自己的答案。
- R 轮结束后，返回收敛得到的答案。

由于成本原因，原始实验使用 N=3、R=2。在困难问题上（MMLU、GSM8K、国际象棋走法合法性判断、人物传记生成），增加智能体数量和轮数都能提升准确率。

跨模型组合优于单模型辩论：ChatGPT + Bard 联合使用 > 任何一个单独使用。

### 稀疏拓扑

"Improving Multi-Agent Debate with Sparse Communication Topology"（arXiv:2406.11776，2024-2025）表明全连接辩论并非总是最优。稀疏拓扑（星形、环形、轴辐式）可以在更低的 token 成本下达到相同准确率。每个辩论者只看到一部分同伴。

由此带来的影响：

- 全连接 N=5、R=3 = 5 × 3 = 15 个提案，每个提案要阅读 4 个同伴 = 60 次批评操作。
- 星形 N=5、R=3（1 个中心 + 4 个分支）= 15 个提案，分支只阅读中心 = 12 次批评操作。

### 辩论何时有用

- **事实性。** N 个独立提案相互核查，减少幻觉。
- **规则遵循。** 国际象棋走法合法性——一个模型漏掉某条规则，其他模型能发现。
- **开放式推理。** 多种思考框架逐步逼近正确答案。

### 辩论何时有害

- **对延迟敏感的用户体验。** N × R 轮串行辩论带来的延迟你可能承受不起。
- **对成本敏感的规模化场景。** 每个问题消耗 N × R 倍的 token。
- **简单的事实查询。** 查一次比辩论五次便宜。

### 2026 年的实际落地形态

- **Anthropic orchestrator-workers**（第 12 课）——带综合步骤的辩论变体之一。
- **LangGraph supervisor**（第 13 课）——中心路由器 + 专家智能体，可以把辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——智能体之间来回交接（handoff），实现迭代式批评。
- **多智能体评估** ——将辩论与 evaluator-optimizer 模式结合，获取评估信号。

### 这个模式哪里会出问题

- **收敛坍缩。** 所有智能体收敛到第一个错误答案上。缓解方法：强制要求若干轮必须存在分歧。
- **中心节点失效。** 在星形拓扑中，一个糟糕的中心节点会污染所有人。轮换中心节点或使用多个中心节点。
- **提示词同质化。** 所有智能体使用相同的提示词，于是产出相同的答案。使用多样化的提示词和/或模型。

## 从零实现

`code/main.py` 用标准库实现了辩论：

- `Debater` 类（脚本化 LLM，每个辩论者带有各自的观点漂移）。
- `FullMeshDebate` 和 `SparseDebate` 两种运行器。
- 三个问题：一个事实型、一个规则型、一个推理型。
- 指标：收敛答案、收敛所需轮数、批评操作总数。

运行：

```
python3 code/main.py
```

输出：各协议的准确率与成本；稀疏拓扑在 3 个问题中有 2 个达到与全连接相同的准确率，且成本更低。

## 生产实践

- **Anthropic orchestrator-workers** 适合简单的 2-3 个 worker 的辩论。
- **LangGraph** 适合带检查点（checkpointing）的有状态多轮辩论。
- **自研方案** 适合研究场景或有特殊正确性保证需求的场景。

## 练习

1. 实现「强制分歧」规则：第 1 轮中每个辩论者必须给出互不相同的提案。测量它对收敛速度的影响。
2. 增加按置信度加权的聚合：辩论者返回 (answer, confidence)；聚合器按置信度加权。有帮助吗？
3. 把其中一个「智能体」换成观点不同的另一个脚本化 LLM。异质性能提升准确率吗？
4. 在你的 3 个问题上测量全连接与稀疏拓扑的 token 成本。绘制成本与准确率的关系图。
5. 阅读 Society of Minds 论文。把你的玩具实现扩展到 N=5、R=3。哪些地方会坏掉？哪些地方会变好？

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|----------------|------------------------|
| 辩论（Debate） | “多智能体批评” | N 个提议者，R 轮交叉批评，最终收敛 |
| 全连接（Full mesh） | “每个人都读所有人” | 每轮中每个辩论者阅读所有同伴 |
| 稀疏拓扑（Sparse topology） | “受限的同伴视野” | 辩论者只阅读一部分同伴 |
| 轴辐式（Hub-and-spoke） | “星形拓扑” | 一个中心辩论者，N-1 个分支只阅读中心 |
| 收敛（Convergence） | “达成一致” | 辩论者收敛到一个共享答案 |
| Society of Minds | “Du 等人的辩论论文” | ICML 2024 多智能体辩论方法 |
