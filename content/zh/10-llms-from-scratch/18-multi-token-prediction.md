# 多 Token 预测（Multi-Token Prediction，MTP）

> 从 GPT-2 到 Llama 3，每一个自回归 LLM 在每个位置上都只用一个损失来训练：预测下一个 token。DeepSeek-V3 在每个位置上增加了第二个损失：预测再下一个 token。这额外的 14B 参数（相对于 671B 的模型）通过梯度流被蒸馏回主模型，而训练好的 MTP 头在推理时被改造成投机解码（speculative decoding）的草稿模型，接受率超过 80%。1.8 倍的生成吞吐量等于白送。本课从 DeepSeek 技术报告出发，构建顺序式 MTP 模块，计算其损失和共享头的参数布局，并解释为什么 MTP 保留了因果链，而 Gloeckle 等人最初的并行 MTP 却破坏了它。

**Type:** Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 10 · 04 (pre-training a mini GPT), Phase 10 · 15 (speculative decoding)
**Time:** ~60 minutes

## 学习目标

- 陈述 MTP 训练目标，并推导跨预测深度的联合损失。
- 解释 Gloeckle 等人的并行 MTP 头（2024）与 DeepSeek-V3 的顺序式 MTP 模块之间的区别，以及顺序式设计为何能保留因果链。
- 计算在预训练中加入 MTP 模块带来的参数与显存开销。
- 从零实现一个 MTP 模块：共享嵌入、按深度的 Transformer 块、投影层和共享输出头。

## 问题背景

下一 token 预测是 LLM 的标准训练目标。每个隐藏状态都被监督去预测且只预测一件事：紧随其后的那个 token。这其实是一个出奇微弱的信号。序列中的大部分信息都延伸到一个 token 之外——结构、连贯性、事实性、算术推理的流程。模型只能在数万亿个 token 上累积大量单 token 信号，才能学会这些东西。

MTP 提出的问题是：如果每个隐藏状态被监督去同时预测多个未来 token 会怎样？Gloeckle 等人（Meta，2024）证明了这是有帮助的。他们的实现是在主干网络之上放置多个独立的输出头，每个头预测一个不同的偏移量。并行、简单，但这些头看到的是同一个隐藏状态，没有任何层次化的精炼——而且预测之间没有因果上的链接，因此无法用于投机解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序式模块，在每个预测深度上保留因果链。模型先从 `h_i^(0)` 预测 `t+1`，再从一个新的隐藏状态 `h_i^(1)` 预测 `t+2`，这个新状态是 `h_i^(0)` 与 `E(t+1)` 嵌入结合的结果，依此类推。每个深度都是一个独立的小型 Transformer 块。共享嵌入和共享输出头使参数开销保持在适度水平。在 DeepSeek-V3 的规模上，MTP 模块在 671B 主模型权重之上额外增加了 14B 参数。这 2% 的开销换来了更密集的训练信号，外加一个推理时现成可用的投机解码草稿模型。

本课将从零构建一个 MTP 模块以及深度为 D 的损失。数学很整洁，实现只有 150 行。

## 核心概念

### 顺序式 MTP 方案

DeepSeek-V3 在主模型之上添加 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度为 `k` 的 token——也就是在给定截至位置 `i` 的前缀时预测 `t_{i+k}`。

模块 `k` 由以下部分组成：

- 一个拥有自己的注意力和 MLP 的 Transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，把上一深度的隐藏状态与下一深度真实 token 的嵌入结合起来。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

训练时，对于截至位置 `i` 的前缀，各深度的隐藏状态为：

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

各深度的预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

各深度的损失是相对于真实 token `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个较小的加权系数——DeepSeek-V3 在训练前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序式而不是并行式

Gloeckle 最初的并行 MTP 有 D 个输出头，每个头直接作用于 `h_i^(0)`。每个头都从同一个主干隐藏状态预测 `t_{i+k}`。这样训练没有问题，但各个预测之间没有相互条件依赖。你无法用 `head_1` 的输出去帮助 `head_2`——这些头是并行触发的。

DeepSeek-V3 的顺序式设计用 `h_i^(k-1)` 加上真实的下一 token 嵌入 `E(t_{i+k})` 来构建 `h_i^(k)`。这保留了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 的模块能看到 `t_{i+k}` 位置上的内容。这在结构上与自回归解码器消费自身输出的方式完全一致——这使得 MTP 模块可以直接用作投机解码的草稿模型。

推理时：把 `h_i^(k-1)` 和已起草的 `t_{i+k}` 喂给模块 `k+1`，得到对 `t_{i+k+1}` 的预测。如此重复。这正是 EAGLE 式的草稿生成，只不过用训练好的 MTP 模块充当草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率超过 80%，整体加速约 1.8 倍。

### 参数核算

对于隐藏维度为 `h`、词表大小为 `V` 的模型：

- 主模型：数十亿参数，外加一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的输出头。无额外参数。
- 共享嵌入：复用主模型的嵌入。无额外参数。
- 每个 MTP 模块：
  - 投影矩阵 `M_k`：`(2h) * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加 MLP（比例为 8/3 的 SwiGLU 通常为 `8h^2`）。每个块约 `12h^2`。

每个模块的额外参数总计约 `~14h^2`。对于 DeepSeek-V3 的 `h = 7168`、D = 1 个模块：理论上是 `~14 * 7168^2 = ~720M` 参数。DeepSeek-V3 报告的是 14B——差距主要来自 MTP 模块中的专家层同样采用了 MoE 结构。

### 投机解码的回报

预训练期间，MTP 模块会使训练变慢约 10%（更多的前向计算，额外的损失）。回报体现在两个方面：

1. 更密集的训练信号。每个隐藏状态获得 D+1 个监督目标。在 MMLU、GSM8K、MATH、HumanEval 上的实测效果：DeepSeek-V3 的消融实验显示稳定的几个百分点的提升。

2. 推理时免费获得投机解码草稿。MTP 模块已经被训练去预测接下来的几个 token。把它改造成草稿网络后，可以达到 80% 以上的接受率。在这个水平上，N=3 或 N=5 的投机解码能带来 1.8 倍的吞吐量。这 10% 的训练时间开销，在你第一次跑推理时就回本了。

### 与 EAGLE 的关系

EAGLE 在预训练结束后单独训练一个小型草稿模型。MTP 则把草稿模型直接融入预训练。两种方法最终收敛到相近的接受率，但流程不同：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3） |
|-----------|---------|------------------|
| 训练时机 | 预训练之后 | 预训练期间 |
| 与现有权重向后兼容 | 是 | 否（需要重新训练） |
| 草稿模型参数 | 1-2 个 Transformer 层 | 1 个 Transformer 块 + 投影层 |
| 接受率 | 0.88-0.92 | 深度 1 时 0.80+ |
| 加速之外的收益 | 仅投机解码 | 更密集的训练信号 + 加速 |

## 从零实现

`code/main.py` 端到端构建一个 MTP 模块：共享嵌入、投影层、Transformer 块、共享输出头。随后在一段较短的合成序列上计算各深度的交叉熵损失，并按组件打印参数量。32 个 token 的玩具词表让数字易于阅读。

### 第 1 步：共享嵌入表

主模型和每个深度的每个 MTP 模块都使用同一张 `vocab_size x hidden` 的表。不是第二份副本——就是字面意义上的同一个张量。

### 第 2 步：按深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # concat along feature dim, then project down to hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # vector addition stand-in
    projected = matvec(M_k, concat)
    return projected
```

真实的 DeepSeek-V3 会把两个经过 RMSNorm 的向量拼接成 `[2h]`，再用一个 `h x 2h` 矩阵投影。玩具实现为了保持标准库的简洁，用向量加法代替。

### 第 3 步：深度 k 处的 Transformer 块

自注意力加 MLP。在玩具实现中，用一个单层线性注意力块和一个 SwiGLU MLP 来呈现结构，且无需 numpy。

### 第 4 步：共享输出头

复用主模型的输出投影。得到词表上的 logits。

### 第 5 步：按深度的损失

用 softmax(logits) 对偏移 `k` 处的真实 token 计算交叉熵。再用 `lambda / D` 缩放因子在各深度间聚合。

### 第 6 步：参数核算

打印总参数量、共享部分（嵌入、输出头）的参数量以及每个模块的额外参数量。展示 MTP 额外参数与主模型规模的比值。

## 生产实践

MTP 已集成进 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列。推理时：

- DeepSeek 自家的服务栈开箱即用地把 MTP 模块当作投机解码器来使用。
- 截至 2026 年 4 月，vLLM 和 SGLang 已有针对 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程给出了一个具体的 MTP 投机解码配置，在 V3 检查点上实测 1.8 倍加速。

何时在新的预训练中使用 MTP：

- 你掌控完整的预训练流水线，想要积累更密集的训练信号。
- 你确定将来要大规模部署该模型，想免费获得投机解码能力。
- 你的隐藏维度至少为 4096。在 1B 规模上，开销带来的损失大于收益。

何时不该使用：

- 微调一个已有的预训练稠密模型。此时 MTP 模块没有经过训练。
- 需要一个干净基线用于对比的研究模型。MTP 会改变模型架构。

1. 运行 `code/main.py`。展示随着合成信号增强，各深度的损失单调下降。把合成数据改成固定模式，并验证深度 1 和深度 2 的损失都能收敛。

2. 计算一个稠密 70B 模型（隐藏维度 8192、80 层）在 D=1 MTP 模块下的参数开销。与 DeepSeek-V3 报告的 14B 开销对比。解释为什么 DeepSeek 的数字更高：MTP 的 Transformer 块继承了相同的 MoE 结构，使每个模块的参数量膨胀。

3. 在玩具实现中实现 D=2：添加第二个 MTP 模块，接收 h^(1) 并预测 `t_{i+2}`。验证联合损失和参数核算与 DeepSeek 论文中的公式 19-21 一致。

4. 把玩具实现改成并行 MTP（Gloeckle 风格）：在主隐藏状态之上添加 D 个输出头，每个头预测一个不同的偏移量。在相同的合成信号上比较各深度损失与顺序式版本的差异。顺序式版本在 k > 1 时应当产生更低的深度 k 损失，因为它以中间预测为条件。

5. 把训练好的 MTP 模块当作 EAGLE 式草稿模型使用：推理时调用模块 k 来提议 `t_{i+k}`。在一段保留序列上测量这些草稿 token 相对于主模型预测的接受率。如果在玩具实现上达到 50% 以上，你就复现了 MTP 作为草稿模型的实证特性。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------------|------------------------|
| MTP 模块 | “额外的损失块” | 一个小型 Transformer 块加投影层，预测主模型前方第 `k` 个位置的 token |
| 预测深度 | “哪个偏移量” | 整数 `k`，使得模块 `k` 从截至位置 `i` 的前缀预测 `t_{i+k}` |
| 并行 MTP | “Gloeckle 风格” | D 个独立的头作用于同一个主干隐藏状态，没有条件链 |
| 顺序式 MTP | “DeepSeek-V3 风格” | 每个模块以上一深度的隐藏状态加下一 token 的嵌入为条件；保留因果链 |
| 共享输出头 | “复用主模型的头” | MTP 模块调用主模型的 LM 头，而不是单独的输出投影 |
| 共享嵌入 | “复用主模型的表” | 同一张词表嵌入表在所有地方使用；没有重复参数 |
| 投影矩阵 M_k | “组合隐藏状态 + 下一 token” | 一个 `h x 2h` 线性层，把上一深度的隐藏状态和目标 token 的嵌入折叠成下一深度的输入 |
| 联合损失 L_MTP | “取平均的额外损失” | 各深度交叉熵损失的算术平均，再按 `lambda` 缩放 |
| 深度 1 接受率 | “MTP 草稿命中的频率” | D=1 MTP 模块的 top-1 预测与主模型 top-1 预测相同的比率；在 DeepSeek-V3 上超过 80% |
| Lambda 加权 | “额外损失的重要性” | 按深度的缩放系数；DeepSeek-V3 在训练初期为 0.3，后期为 0.1 |
