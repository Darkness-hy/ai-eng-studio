# 从 CLIP 到 BLIP-2 —— Q-Former 作为模态桥梁

> CLIP 能对齐图像和文本，但无法生成描述、回答问题或进行对话。BLIP-2（Salesforce，2023）用一座小巧的可训练桥梁解决了这个问题：32 个可学习查询向量通过交叉注意力关注冻结 ViT 的特征，然后直接插入冻结 LLM 的输入流。1.88 亿参数的桥梁把一个 11B 的 LLM 和一个 ViT-g/14 连接了起来。2026 年之前所有基于适配器的视觉语言模型（VLM）—— MiniGPT-4、InstructBLIP、LLaVA 的各路近亲 —— 都是它的后裔。本课将解读 Q-Former 的架构，讲解它的两阶段训练，并构建一个玩具版本，把视觉 token 输入到一个冻结的文本解码器中。

**Type:** Build
**Languages:** Python (stdlib, cross-attention + learnable-query demo)
**Prerequisites:** Phase 12 · 02 (CLIP), Phase 7 (Transformers)
**Time:** ~180 minutes

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的 LLM 之间放一个可训练瓶颈，在成本和稳定性上都优于端到端微调。
- 实现一个交叉注意力块，让一组固定的可学习查询向量关注外部图像特征。
- 梳理 BLIP-2 的两阶段预训练：先表征学习（ITC + ITM + ITG），再生成学习（使用冻结解码器的 LM 损失）。
- 将 Q-Former 与 LLaVA 使用的更简单的 MLP 投影器进行对比，并论证各自在什么情况下更优。

## 问题背景

你手里有一个冻结的 ViT，每张图像产生 256 个维度为 1408 的图块（patch）token。你还有一个冻结的 7B LLM，它期望接收维度为 4096 的 token 嵌入。最直接的桥梁 —— 一个从 1408 到 4096 的线性层 —— 是可行的，但把全部 256 个图块 token 喂进 LLM 的上下文，每张图像要额外消耗 256 个 token。一个 32 张图像的批次，仅视觉模态就要吃掉 8192 个 token。

BLIP-2 提出的问题是：能否把 256 个 token 的图像表示压缩到少得多的 token 数（比如 32 个），同时保留足够的信息，让 LLM 能够生成描述、回答问题并对图像进行推理？而且，能否在完全不动冻结主干的前提下训练这座桥梁，把训练成本控制在桥梁自身的参数量上？

答案就是 Q-Former。32 个可学习的「查询」向量对 ViT 的图块 token 做交叉注意力，产出一个 32 token 的视觉摘要供 LLM 消费。总共 1.88 亿参数。在接触 LLM 之前，先用对比、匹配和生成三种目标进行训练。

## 核心概念

### 可学习查询

Q-Former 的核心技巧：不让 LLM 的文本 token 去关注图像图块，而是引入一组全新的 32 个可学习查询向量 `Q`，让*它们*去关注图像图块。这些查询是模型的参数 —— 在训练中学得，并且每张图像都使用同一组 32 个查询。

经过交叉注意力之后，每个查询都持有图像的一份压缩摘要 ——「描述主体对象」「描述背景」「数一数物体」等等。这些查询并不会真的按语义标签分工；它们学到的是任何能让下游损失下降的编码方式。

### 架构

Q-Former 是一个小型 Transformer（12 层，约 1 亿参数），包含两条通路：

1. 查询通路：32 个查询向量先经过自注意力（彼此之间），再对冻结 ViT 的图块 token 做交叉注意力，最后通过 FFN。
2. 文本通路：一个类 BERT 的文本编码器，与查询通路共享自注意力和 FFN 权重。文本通路的交叉注意力被禁用。

训练时两条通路同时运行。查询和文本通过共享的自注意力交互，这意味着在需要的任务上（ITM、ITG），查询可以以文本为条件。推理时向 VLM 交接的阶段，只有查询通路在运行，产出 32 个视觉 token。

### 两阶段训练

BLIP-2 的预训练分两个阶段：

第一阶段：表征学习（不涉及 LLM）。三个损失：

- ITC（图文对比，image-text contrastive）：在池化后的查询 token 与文本 CLS token 之间做 CLIP 风格的对比学习。
- ITM（图文匹配，image-text matching）：二分类器 —— 这对图文是否匹配？使用难负例挖掘。
- ITG（基于图像的文本生成，image-grounded text generation）：以查询为条件，对文本施加因果 LM 头。迫使查询编码出可用于文本生成的内容。

只训练 Q-Former。ViT 保持冻结。不涉及任何 LLM。

第二阶段：生成学习。接上一个冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小线性层把 32 个查询输出投影到 LLM 的嵌入维度。将它们拼接在文本提示之前。在「提示 + 图像 + 描述」拼接序列上的 LM 损失下，只训练这个线性投影和 Q-Former。

第二阶段结束后，Q-Former + 投影层就是完整的视觉适配器。推理流程：图像 → ViT → Q-Former → 线性投影 → 拼接在文本之前 → 冻结 LLM 产出输出。

### 参数经济学

BLIP-2 配置为 ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，训练）= 总计 8B，仅训练 188M。Q-Former 只占整个模型栈参数量的约 2.4%。训练成本也随之体现：几块 A100 跑几天，而端到端训练要数周。

质量方面：BLIP-2 在零样本 VQA 上追平甚至超过 Flamingo-80B，而体量只有后者的五十分之一。这座桥是有效的。

### InstructBLIP 与指令感知的 Q-Former

InstructBLIP（2023）为 Q-Former 增加了一个额外输入：指令文本本身。在交叉注意力时，查询现在可以同时访问图像图块和指令。查询可以按指令分化（「数一数车辆」「描述氛围」），而不是只学一份固定的摘要。在留出任务上取得了基准测试提升。

### MiniGPT-4 与仅训练投影器的方案

MiniGPT-4 保留了 Q-Former，但只训练输出线性投影，其余全部冻结。便宜，但代价是质量 —— 那些查询是 BLIP-2 的，不是你自己的。适合快速迭代，但不是最佳架构。

### 为什么 LLaVA 走了更简单的路

LLaVA（2023，第 12.05 课）用一个普通的 2 层 MLP 取代了 Q-Former，把每一个 ViT 图块 token 都投影到 LLM 空间 —— 24x24 网格下每张图像 576 个 token，全部喂给 LLM。压缩率更差，但让 LLM 可以直接关注原始图块。当时这是有争议的；到 2023 年末它成为主流，因为视觉指令数据（LLaVA-Instruct-150k）证明了 MLP 经过训练足以保留充分的信号。代价是：LLaVA 的上下文消耗更快，但它能自然地扩展到多图和视频场景。

到 2026 年，领域出现了分化：在 token 预算受限的场景（长视频、多图像）Q-Former 依然存活；在追求单 token 原始质量的场景 MLP 投影器占据主导。

### 门控交叉注意力：Flamingo，共同的祖先

Flamingo（第 12.04 课）早于 BLIP-2，使用了同样的交叉注意力思想，但作用在冻结 LLM 的每一层，而不是作为单一桥梁。BLIP-2 证明了只压缩到输入层依然可行。Gemini 和 Idefics 则将两者结合：交错的输入 token，加上可选的门控交叉注意力以支持上下文内少样本学习。

### 2026 年的后裔们

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4，以及出于 token 预算考虑的大多数视频-语言模型。
- Perceiver 重采样器：Flamingo 的变体（第 12.04 课）；Idefics 系列、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

四种方案都是合理的。决定性问题在于：你受限的是 token 预算，还是单 token 质量。

## 生产实践

`code/main.py` 用纯标准库构建了一个 Q-Former 风格的交叉注意力：

1. 模拟 256 个图像图块 token（维度 128）。
2. 实例化 32 个可学习查询（维度 128）。
3. 运行缩放点积交叉注意力（Q 来自查询，K/V 来自图块）。
4. 通过一个线性层投影到 LLM 维度（512）。
5. 输出 32 个可直接供 LLM 使用的视觉 token。

所有数学运算都用纯 Python 实现（对向量做嵌套循环）。虽是玩具，但形状正确。注意力权重矩阵会被打印出来，你可以看到每个查询从哪些图块提取了信息。

2. 在 BLIP-2 第一阶段中，Q-Former 同时运行三个损失：ITC、ITM、ITG。用伪代码写出每个损失的前向函数签名。哪一个要求文本编码器通路处于激活状态？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| Q-Former | 「查询 Transformer」 | 带有 32 个可学习查询向量的小型 Transformer，通过交叉注意力关注冻结 ViT 的特征 |
| 可学习查询 | 「视觉版软提示」 | 一组固定参数，充当交叉注意力的查询侧；按模型学习，所有输入共享 |
| 交叉注意力 | 「Q 来自这边，K/V 来自那边」 | 查询、键、值来自不同来源的注意力机制；查询正是借此从 ViT 图块中提取信息 |
| ITC | 「图文对比」 | 在 Q-Former 池化查询与文本 CLS 之间施加的 CLIP 风格损失 |
| ITM | 「图文匹配」 | 在难负例挖掘的样本对上做二分类；迫使查询辨别细粒度的不匹配 |
| ITG | 「基于图像的文本生成」 | 以查询为条件生成文本的因果 LM 损失；迫使查询编码可被文本解码的内容 |
| 两阶段预训练 | 「先表征，后生成」 | 第一阶段单独训练 Q-Former（ITC/ITM/ITG）；第二阶段接上冻结 LLM，只训练投影层 + Q-Former |
| 冻结主干 | 「不要微调」 | 视觉编码器和 LLM 权重固定不动；只训练桥梁 |
| 投影头 | 「线性映射到 LLM 维度」 | 把 Q-Former 输出映射到 LLM 嵌入维度的最后一个线性层 |
| Perceiver 重采样器 | 「Flamingo 的版本」 | 类似的可学习查询交叉注意力，Flamingo 在每一层都使用，而非只作单一桥梁 |
