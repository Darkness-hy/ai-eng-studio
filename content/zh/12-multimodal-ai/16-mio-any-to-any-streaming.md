# MIO 与任意到任意（Any-to-Any）流式多模态模型

> GPT-4o 交付了一个大多数开源模型无法复刻的产品：一个能听语音、看视频并实时用语音回应的智能体。到 2024 年底，开源生态的回应是 MIO（Wang et al., 2024 年 9 月）。MIO 把文本、图像、语音和音乐统统分词化，在交错排列的序列上训练一个因果 Transformer，从而实现任意模态到任意模态的生成。AnyGPT（Zhan et al., 2024 年 2 月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI, 2023 年 12 月）则是带视觉与动作落地能力的近亲。本课解读这套任意到任意的模式——四个分词器、一个 Transformer、对流式友好的解码。

**Type:** Learn
**Languages:** Python (stdlib, four-modality token allocator + streaming decode loop)
**Prerequisites:** Phase 12 · 11 (Chameleon), Phase 6 (Speech and Audio)
**Time:** ~120 minutes

## 学习目标

- 设计一个共享词表，让文本、图像、语音和音乐 token 互不冲突地共存。
- 在压缩率与重建质量的权衡上，比较 SEED-Tokenizer（图像）与 SpeechTokenizer 残差向量量化（speech residual-VQ）。
- 解释逐步构建任意到任意生成能力的四阶段训练课程。
- 说出三个开源任意到任意方案及其主要取舍：MIO、AnyGPT、Unified-IO 2。

## 问题背景

统一多模态模型说起来容易，规模化做出来很难。在 2024 年之前，大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一跳都会丢失信息、增加延迟，并让训练更加复杂。GPT-4o 的演示视频展示了一种单模型方案，响应在一秒以内；开源系统落后了好几个月。

工程上的挑战：

- 每种模态都必须有分词器，压缩后要"足够无损"以支持重建，且产出 token 的速率必须是 Transformer 消化得了的。
- 一个词表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。至少四万多个条目。
- 训练数据必须覆盖每一种输入-输出组合（文本→图像、图像→语音、语音→图像等），否则模型必须靠组合泛化。
- 推理必须以足够快的速度流式输出 token，才能达到对话级延迟（首个音频字节 <500ms）。

## 核心概念

### 四种模态，四个分词器

MIO 的分词器组合：

- 文本：标准 BPE，词表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图 32x32 个 token。
- 语音：SpeechTokenizer 残差向量量化（residual-VQ, 2023）——将 16kHz 波形编码为 8 个层级化码本；第一层是粗粒度内容，后续层级补充韵律和说话人特征。
- 音乐：类似的 residual-VQ（Meta 的 MusicGen / Encodec 家族），4-8 个码本。

每种模态都产出整数 token。这些 token 在共享词表中占据互不重叠的 ID 区间：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

总计约 48k 词表。输入嵌入和输出投影覆盖全部区间。

### 流式解码

语音生成使用 residual-VQ。Transformer 预测基础层（第 0 层）语音 token；一个并行解码的残差量化器预测后续层级。每个第 0 层 token 大约对应 16kHz 下 50ms 的音频。

流式处理模式：

1. 用户对麦克风说话；实时音频分词器每 50ms 输出一批语音 token。
2. MIO 边到边吃，token 到达即消费（提示词预填充 + 增量前向计算）。
3. 输出 token 边生成边流出；一个并行的语音解码器以约 50-150ms 的延迟把它们转换成音频采样。
4. 首个音频字节耗时：MIO 论文中约为 300-500ms，逼近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。其中 Moshi 在单张 GPU 上做到了 160ms 的往返延迟。

### 四阶段训练课程

MIO 的训练课程：

1. 阶段 1——对齐。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每个模态对使用各自的 token 词表区段。训练共享词表。
2. 阶段 2——交错。多模态交错文档（带图片和视频的博客、带文字稿的播客等）。训练跨模态上下文能力。
3. 阶段 3——语音增强。补充额外的音频数据来提升语音质量，同时不损失文本能力。
4. 阶段 4——SFT。跨模态指令微调：VQA、图像描述、旁白生成、语音到语音对话。

缺少任何一个阶段都会损害特定能力：跳过阶段 2，模型就失去跨模态上下文能力；跳过阶段 3，语音质量就会很差。

### 视觉思维链

MIO 引入了视觉思维链（chain-of-visual-thought）：模型把生成中间图像 token 作为推理步骤。对于"猫在爬树吗？"这个问题，模型会：

1. 输出 `<image>` token，渲染出场景（来自输入图像或一张草图）。
2. 输出分析这张草图的文本。
3. 输出最终答案。

渲染出的中间图像充当草稿板。空间推理任务上的基准成绩因此提升。这个思路与文本推理中的思维链（chain-of-thought）如出一辙。

### 任意到任意赛道的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：增加了视觉动作输出、深度、法线。任务更多样，规模更小。
- NExT-GPT（arXiv:2309.05519）：LLM + 各模态专属的扩散解码器。不是单模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜空间实现任意到任意。

MIO 最接近纯 token 的任意到任意方案。AnyGPT 是它的概念先驱。

### 延迟预算

对一个对话式产品来说，每个组件的延迟都很关键：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史记录）：8B 模型上约 100ms。
- 首个输出 token：约 50ms。
- 并行 residual-VQ + 语音解码器：约 100-150ms。

首个音频字节总耗时：最低约 300ms。GPT-4o 宣称约 250ms。Moshi 宣称 160ms。根据公开基准，MIO/AnyGPT 处于 400-600ms 区间。

### 任意到任意为何依然困难

即使到了 2026 年，开源任意到任意模型在两个维度上仍落后于闭源模型：

- 语音质量。residual-VQ 分词器是有损的；与 ElevenLabs 级别的音色相比，对话语音听起来机械生硬。
- 跨模态推理。让模型"把你看到的东西唱出来"，失败率仍然高于纯视觉任务。

这些都是尚未解决的研究问题。Qwen3-Omni（第 12.20 课）是 2025 年最先进的开源尝试。

## 生产实践

`code/main.py`：

- 定义四模态词表分配方案并打印出来。
- 把一组多模态输入（文本、图像、音频片段、音乐）路由到分词器路由器。
- 模拟一次文本到语音回复的流式解码，并统计延迟。
- 在给定编码器、预填充和解码器延迟的条件下，计算预期的首个音频字节耗时。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标应该是多少？列出消耗时间的各个组件。

2. SpeechTokenizer residual-VQ 使用 8 个码本。请论证为什么残差层必须并行解码（而非顺序解码），以及这能节省多少延迟。

3. 你的词表有 32k 文本 + 4k 图像 + 4k 语音。再加入 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数开销是多少？

4. 视觉思维链会输出一张中间图像。哪些类型的问题会因此受益？哪些类型会因为额外的 token 而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述它的"内心独白"（inner monologue）技术，并与 MIO 的视觉思维链作比较。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|-----------------|------------------------|
| 任意到任意（Any-to-any） | "多模态输入/输出" | 一个能在任意方向上接受和输出文本、图像、语音、音乐的单一模型 |
| 残差向量量化（Residual-VQ） | "语音分词器堆栈" | 多码本分词方案，每一层补充信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | MIO 使用的离散图像分词器，码本含 4096 个条目 |
| 视觉思维链（Chain-of-visual-thought） | "视觉草稿板" | 模型在给出最终答案前生成一张中间图像作为推理步骤 |
| 首个音频字节耗时（Time-to-first-audio-byte） | "TTFAB" | 从用户发声到输出第一个音频字节的延迟；<500ms 才有对话感 |
| 四阶段训练课程（Four-stage curriculum） | "训练配方" | 按顺序依次为：对齐 -> 交错 -> 语音增强 -> SFT |
