# InternVL3：原生多模态预训练

> 在 InternVL3 之前，所有开源 VLM 都遵循同一套三步配方：拿一个在数万亿文本 token 上训练好的文本 LLM，外挂一个视觉编码器，再对接缝处做微调。这种做法可行，但会积累对齐债（alignment debt）——文本 LLM 的全部预训练预算都花在了纯文本上，并不天然理解视觉 token。当你事后补加视觉能力时，LLM 必须在不遗忘文本能力的前提下，重新学习如何把视觉输入与其文本推理关联起来。InternVL3（Zhu et al., 2025 年 4 月）拒绝了这种事后补加的路线：只跑一次预训练，从第一步起就将文本与多模态数据交错训练。结果是 78B 参数的开源模型在 MMMU-Pro 上追平了 Gemini 2.5 Pro。本课带你读懂原生预训练的论证逻辑，以及采用它之后会发生哪些变化。

**Type:** Learn
**Languages:** Python (stdlib, training-corpus mixer)
**Prerequisites:** Phase 12 · 05, Phase 12 · 07 (recipes)
**Time:** ~120 minutes

## 学习目标

- 解释为什么事后补加（post-hoc）的 VLM 训练会积累对齐债，并列举三种可测量的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 原生预训练语料的配比，以及文本 : 交错 : 图文配对的比例为何重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（Visual Resolution Router, ViR）和视觉-语言解耦部署（Decoupled Vision-Language, DvD）这两项部署优化。

## 问题背景

事后补加是 VLM 训练的默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——全都是拿一个已预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）再加上视觉能力。训练阶段通常是这样的：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在图文配对数据上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的任务特定微调。

对齐债会表现出三种症状：

- 灾难性遗忘。事后补加的 VLM 会遗忘纯文本能力。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本智能体的表现倒退。
- 答案漂移。同一个视觉问题换种细微措辞就得到不同答案。视觉编码器与 LLM 之间的绑定强度，弱于 LLM 自身 token 之间的绑定。
- 视觉-文本不一致。VLM 可以正确描述一张图像，然后在回答问题时却与自己的描述自相矛盾。视觉 token 没有像文本那样参与 LLM 的内部一致性校验。

这些症状有充分的文献记录。MM1.5 第 4 节给出了量化数据。LLaVA-OneVision 的消融实验也有所暗示。原生预训练就是答案。

## 核心概念

### 原生多模态预训练

InternVL3 从零开始训练，语料从第一步起就是原生多模态的。配比为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 图文交错数据（OBELICS、MMC4 风格）
- 20% 图像-描述配对数据
- 5% 视频-文本数据

视觉 token、文本 token 以及跨模态交互，从第一个梯度步开始就参与同一个损失。没有对齐预训练阶段，没有投影器冻结阶段，也没有需要事后弥补的灾难性遗忘。

基座模型的训练是单阶段的。之后再做指令微调，但基座模型已经把视觉 token 当作一等公民来理解了。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：位置编码按模态类型（文本、图像、视频）变化，并带有可学习的缩放。具体而言：

- 文本 token 使用 1D 位置（文本索引）。
- 图像 patch 使用 2D 位置（行、列）。
- 视频帧使用 3D 位置（时间、行、列）。

三者共享同一个 RoPE 频率基数，但每个频带的隐藏维度分配是可学习参数，而非固定切分。这让模型在预训练期间可以自由权衡时间分辨率与空间分辨率。

V2PE 的消融实验结论：在相同算力下，视频基准上比 M-RoPE 高 1-2 分。算不上革命性，但更干净。

### 视觉分辨率路由器（ViR）

这是一项部署优化。并非所有图像都需要全分辨率编码。一张细节稀少、只有一个物体的照片，若按 1280px 原生分辨率编码就是浪费 token。ViR 是一个小型分类器，在编码之前预测回答该问题所需的最低分辨率。

路由分为三档：低分辨率（256 token）、中等（576）、高（2048+）。在生产流量中，60% 的查询用低或中档就足够了。净效果：质量不变，吞吐量提升 2-3 倍。

### 视觉-语言解耦部署（DvD）

部署大型 VLM 时，视觉编码器对每张图像只运行一次，而 LLM 要为每个输出 token 自回归地运行。两个组件的瓶颈不同（视觉 = 卷积 + 注意力的 GPU 显存带宽；LLM = KV 缓存）。DvD 将二者拆到不同 GPU 上，中间以流式传输衔接。

对于一个 8B + 400M 编码器的模型，DvD 相比同卡共置部署，单节点吞吐量大约翻倍。

### 单阶段 vs 多阶段的质量对比

InternVL3 的核心基准主张：78B 参数时，MMMU-Pro 追平 Gemini 2.5 Pro；38B 时追平 GPT-4o；8B 时领跑开源 8B 榜单。这一切都建立在单阶段预训练 + 指令微调的配方之上。

对齐债假说是可测量的：每获得一单位视觉基准提升，InternVL3-8B 在文本基准（MMLU、GSM8K）上掉的分比 Qwen2.5-VL-7B 更少。这个模型更像一个通才，因为训练是一个整体，而不是两段拼接。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）放大了这套配方。同样的原生预训练路线，更多数据，更多参数。MMMU 上的提升是渐进式的。

InternVL-U（2026 年）增加了统一生成能力——在同一骨干网络上通过 MMDiT 头输出图像。"U" 代表 "Understanding + generation"（理解 + 生成），追随 Transfusion 风格的统一模型（见第 12.13 课）。同一个原生预训练骨干同时支撑理解头和生成头。

### 原生预训练的取舍

原生预训练并非没有代价：

- 算力。从零训练一个新 VLM 的成本与训练一个文本 LLM 相当——数百万 GPU 小时。事后补加可以复用现有 LLM 权重，省下大部分成本。
- 数据。大规模图文交错语料很稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。而纯文本数据动辄 15T token。多模态预训练数据的稀缺是一个硬约束。
- 基座 LLM 复用。原生预训练放弃了日后换装新 LLM 的选项。事后补加路线只需重训适配器，就能把 Llama-3.1 换成 Llama-4。

InternVL3 押的注是：对齐债的代价比放弃复用更大。基准测试支持这一主张。但高昂的训练成本让后来的实验室无法低成本复制。事后补加的 VLM 仍会继续存在，因为对大多数项目而言它们依然更便宜。

## 生产实践

`code/main.py` 是一个训练语料配比器兼 ViR 路由模拟器。它会：

- 接收目标语料配比（%文本、%交错、%图文配对、%视频），计算每种模态的预期训练步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节），并报告平均 token 数。
- 根据编码器与 LLM 的 FLOPs 给出 DvD 吞吐量估算。
- 打印事后补加与原生预训练在参数量、算力、数据和预期对齐债症状上的并排对比。

## 交付产物

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一份拟议的 VLM 训练方案，它会审计该走原生路线还是事后补加路线，标记对齐债风险，并推荐语料配比。当你在规划一个新的开源 VLM 项目、需要选定训练策略时，就用它。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后补加）之间的算力差距。GPU 小时的比值大约是多少？是什么造成了这个差距？

2. InternVL3 报告的配比是 40% 文本 / 35% 交错 / 20% 图文配对 / 5% 视频。如果你的目标任务以视频为主，提出一个新配比，并论证为什么基座模型仍然需要大量文本和图文配对数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。指出事后补加训练倒退最严重的具体基准是哪个。倒退的代价有多大？

4. ViR 把 60% 的流量路由到低分辨率编码。哪些类型的查询会被它错误路由（本需要高分辨率却被送去低分辨率）？提出三种路由器失效模式。

5. DvD 把视觉和 LLM 拆到不同 GPU 上。在什么样的流量模式下，DvD 反而会降低吞吐量？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|-----------------|------------------------|
| 原生多模态预训练 | "从零一起训" | 文本 + 图像 + 视频 token 从第 1 步起就参与损失，而不是事后外挂 |
| 对齐债 | "事后补加的代价" | 把视觉外挂到冻结 LLM 上所导致的、可测量的文本能力倒退和答案一致性下降 |
| V2PE | "可变视觉位置编码" | 按模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继任方案 |
| ViR | "分辨率路由器" | 在编码前为每个查询选择所需最低分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一块 GPU，LLM 在另一块，中间流式交接；让大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年的后续版本，在原生预训练骨干上增加图像生成头 |
| 交错语料 | "OBELICS / MMC4" | 文本与图像按自然阅读顺序排列的文档；原生预训练的原材料 |
