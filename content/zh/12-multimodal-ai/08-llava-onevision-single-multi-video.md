# LLaVA-OneVision：单图、多图、视频统一于一个模型

> 在 LLaVA-OneVision（Li et al.，2024 年 8 月）之前，开源 VLM 世界存在彼此独立的几条谱系：处理单图的 LLaVA-1.5，处理多图的 Mantis 和 VILA，处理视频的 Video-LLaVA 和 Video-LLaMA。每个模型都在自己的基准上称王，却在其他场景上失手。LLaVA-OneVision 提出：一套课程式训练（curriculum）就能让一个模型在全部三种场景上称霸，而且由此涌现的任务迁移效应（单图能力迁移到视频、多图推理迁移到单图）胜过各路专家模型的简单加和。这套配方简单得出人意料：一个在各场景间保持恒定的视觉 token 预算，加上一条从单图到 OneVision（多图）再到视频的显式课程。本课将逐一拆解这个预算、这条课程以及那些涌现行为。

**Type:** Build
**Languages:** Python (stdlib, token budget solver + curriculum planner)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 12 · 06 (any-resolution)
**Time:** ~180 minutes

## 学习目标

- 设计一个在单图、多图、视频输入之间保持恒定的视觉 token 预算。
- 编排一条能把能力从单图迁移到视频而不发生灾难性遗忘的训练课程。
- 解释为什么在课程设计得当时，同等参数量下单一模型能战胜各场景的专家模型。
- 说出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、set-of-mark 提示、iPhone 截图智能体。

## 问题背景

图像、多图和视频对模型的压力点各不相同。

单图需要高分辨率 token（AnyRes，约 2880 个视觉 token）来捕捉 OCR 和细粒度细节。每个样本的预算：一张图像，2880 个 token。

多图需要若干张中等分辨率的图像（每张约 576 个 token），这样跨图推理才能装进上下文。每个样本的预算：4-8 张图像，每张 576 个，共 2300-4600 个 token。

视频需要大量低分辨率帧（池化后每帧约 196 个 token）来捕捉时序动态。每个样本的预算：8-32 帧，每帧 196 个，共 1600-6200 个 token。

如果训练多个独立模型，你只需选定一种预算。如果训练单一模型，就需要预算在各场景间合理伸缩而不撑爆上下文。

在 OneVision 之前，默认答案是「只训一种场景，无视其余」。Video-LLaVA 通过额外训练阶段把视频能力补装到图像模型上。LLaVA-NeXT 用分块（tiling）增加了多图支持。没有一个能干净地同时处理三种场景。

## 核心概念

### OneVision 的 token 预算

LLaVA-OneVision 选定了一个统一的视觉 token 预算，每个样本约 3000-4000 个 token，按场景做不同分配：

- 单图：AnyRes-9（3x3 个分块 + 缩略图），每个分块 384 分辨率、729 个 patch，再做激进的 2x2 双线性池化 → 每块 182 个。总计：9 * 182 + 182 = 1820 个 token。或者用每块 729 的 AnyRes-4 = 2916 + 729。
- 多图：每张图像中等分辨率（384，不分块），729 个 token，不做池化。预算 6 张图 → 4374 个 token。
- 视频：32 帧，384 分辨率，做激进的 3x3 双线性池化 → 每帧 81 个 token。总计：32 * 81 = 2592 个 token。

这套分配让总 token 数大致保持恒定。LLM 永远不会遇到撑爆上下文的批次。编码器在不同场景下产生不同的几何形状，但 LLM 消耗的预算始终相同。

### 三阶段课程

LLaVA-OneVision 的训练分三个阶段：

1. 单图 SFT（SI 阶段）。所有数据都是单图加文本。以高分辨率 AnyRes 输入训练。这一阶段教会模型感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 专门的单图数据。
2. OneVision SFT（OV 阶段）。混合单图 + 多图 + 视频（帧均匀采样）。在统一 token 预算下训练。这一阶段教会模型处理异构的批次形状。不重置权重——直接从 SI 阶段继续。
3. 任务迁移（TT 阶段）。在目标任务混合数据上继续训练，通常依产品需求加重多图或视频的比例。这是面向部署的可选微调。

关键在于：课程顺序至关重要。先训视频或先训多图，即便数据完全相同，图像性能也比先训单图更差。论文对此做了显式消融。

### 课程为何有效

单图训练打下感知基础。patch token 承载细粒度视觉特征；LLM 学会将它们与文本融合。多图和视频引入的是结构性挑战（哪张图是哪张、什么先发生），缺乏扎实的感知基础时很难学会。

如果从零开始把所有场景混在一起训练，模型会在感知上欠拟合（每个批次里单图数据有限），在结构上过拟合（多图/视频数据很多）。结果是：一个会照搬跨图推理套路、但视觉理解肤浅的模型。

课程式排序让你先从 SI 阶段获得感知能力，再从 OV 阶段获得组合/时序推理能力，两者都不丢失。

### 涌现的跨场景能力

LLaVA-OneVision 论文报告了三种涌现能力：

1. 多摄像头推理。训练时多图和视频是分开的；推理时被要求对一个多摄像头的驾驶场景做推理。模型正确整合了各个视角，尽管训练中从未见过这种确切格式。
2. Set-of-mark 提示。用户在图像中用数字标记标注物体；模型能推理「标记 3 相对标记 7 在做什么」。训练数据里既没有标记也没有标注；这一能力来自空间定位 + 多图指代的组合。
3. iPhone 截图智能体。用户提供一张 iPhone 屏幕截图并要求规划下一步点击。训练数据包含 UI 截图、用户操作流程的视频、多图的前后对照对。模型泛化到了智能体用例。

这些都不是训练过的任务；它们从课程的组合式结构中涌现出来。

### 视觉 token 池化

token 预算要求做池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 个 patch 变成 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化在 patch 网格空间而非 token 空间进行，以保留局部性。

每个场景的池化因子本身就是一个超参数。池化越少 = token 越多 = 表征越丰富。池化越多 = token 越少 = 能塞下更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续工作（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上「完全开放」。它在部分基准上追平了与专有模型的差距，并让这套配方人人可用。课程不变、数据更多、基座 LLM 更好。架构没有改动。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（第 12.09 课）做出了不同的选择。它使用 M-RoPE 和动态 FPS，而不是固定池化。它的预算随输入伸缩——1 分钟的视频比 5 秒的视频用更多 token。LLaVA-OneVision 则固定预算、调整池化。两者都行得通；它们在可配置性与可预测性之间各取一端。

## 生产实践

`code/main.py` 是一个面向 OneVision 风格 VLM 的课程与预算规划器。给定每个样本的 token 预算和目标场景配比（比如 40% 单图、30% 多图、30% 视频），它会：

- 为每个场景分配分辨率、池化因子和帧数。
- 检查每个场景是否都能装进共享预算。
- 报告预期 token 数、LLM FLOPs，以及哪些场景的 token 配额不足。
- 打印逐阶段的训练计划。

可以用它来规划一次 OneVision 微调，或对 VLM 部署的单请求成本做合理性检查。

## 交付产物

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每样本预算，它输出 AnyRes 因子、每帧池化、视频帧数和课程各阶段权重。每当你训练或微调一个统一场景的 VLM 时，都可以使用它。

## 练习

1. 你的产品需要支持 80% 单图、10% 多图（2-4 张）、10% 视频（8-16 帧）。设计 token 预算。从不做重度多图省下的预算，你会用在哪里？

2. 阅读 LLaVA-OneVision 论文第 4.3 节（涌现能力）。提出第四种这套课程很可能解锁、但论文没有报告的涌现能力。

3. 调换课程顺序——先训多图，再训单图，最后训视频。预测哪些基准会下降，并解释原因。

4. 论文报告的视频基准每个样本只用 8 帧训练。这能泛化到推理时的 30 秒视频吗？先崩的是 token 预算还是时序推理？

5. 把 24x24 个 patch 双线性池化到 12x12，每个维度缩减 4 倍。用 Python 标准库实现这个池化，并验证每个 2x2 块的均值与双线性输出一致。

## 关键术语

| 术语 | 大家怎么说 | 实际含义 |
|------|-----------------|------------------------|
| OneVision 场景 | 「单图、多图或视频」 | 统一 VLM 处理的三种输入形态之一；预算在各场景间保持恒定 |
| token 预算 | 「每个样本多少 token」 | LLM 在每个训练/推理样本中看到的视觉 token 总数，通常为 3000-4000 |
| 课程（Curriculum） | 「训练顺序」 | 为了涌现式迁移而选定的阶段顺序（单图 → 多图 → 视频） |
| 双线性池化 | 「token 压缩」 | 对（2D）patch 网格做双线性插值以减少 token 数，同时保留局部性 |
| 涌现能力 | 「没训过，照样行」 | 在没有匹配训练数据的情况下于推理时出现的能力，源于课程的组合效应 |
| AnyRes-k | 「k 分块配置」 | k 个固定分辨率的子分块加一张缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | 「跨场景泛化」 | 通过共享骨干网络，在单图上学到的能力可用于视频（反之亦然） |
