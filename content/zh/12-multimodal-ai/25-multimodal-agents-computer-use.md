# 多模态智能体与计算机操作（毕业项目）

> 2026 年的前沿产品是这样一种多模态智能体：它能读取屏幕截图、点击按钮、浏览网页界面、填写表单，并端到端地完成整个工作流。SeeClick 和 CogAgent（2024）验证了 GUI 定位（GUI grounding）这一基础能力。Ferret-UI 把它扩展到了移动端。ChartAgent 为图表理解引入了视觉工具调用。VisualWebArena 和 AgentVista（2026）是前沿模型竞相追逐的基准——即便是 Gemini 3 Pro 和 Claude Opus 4.7，在 AgentVista 的困难任务上也只有约 30% 的得分。这个毕业项目把第 12 阶段的所有线索汇于一处：感知（高分辨率 VLM）、推理（带工具调用的 LLM）、定位（坐标输出）、长程记忆与评估。

**Type:** Capstone
**Languages:** Python (stdlib, action schema + agent loop skeleton)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 12 · 09 (Qwen-VL JSON), Phase 14 (Agent Engineering)
**Time:** ~240 minutes

## 学习目标

- 设计多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建一个 GUI 定位输出 schema（点击坐标、输入文本、滚动、拖拽），让 VLM 以 JSON 形式输出。
- 对比纯截图智能体、无障碍树（accessibility-tree）智能体与混合智能体。
- 在一个小规模 VisualWebArena 切片上搭建多模态智能体基准评估。

## 问题背景

一个订票网站的工作流："帮我找一班 4 月 15 日飞东京的航班，靠过道座位，800 美元以内，订下来。"

多模态智能体需要：

1. 对浏览器进行截图。
2. 把截图 + URL + 目标解析成一个计划。
3. 输出一个结构化动作：点击（在坐标 x,y）、输入 "Tokyo"（在元素 E）、向下滚动、选择（单选按钮）。
4. 把该动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复，直到任务完成。

每一步都是一次多模态 VLM 调用。VLM 的输出必须是可解析的 JSON。错误会随步骤累积放大，所以错误恢复至关重要。

## 核心概念

### GUI 定位——基础能力

GUI 定位指的是：给定一张屏幕截图和一条自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是第一个开放的规模化成果：在合成 + 真实 GUI 数据上微调一个 VLM，把坐标作为普通文本 token 输出。这个方法是可行的。

CogAgent（arXiv:2312.08914）针对密集 UI 增加了 1120x1120 的高分辨率编码。成绩：在网页导航上约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动端 UI，并与 iOS 无障碍数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 字段有助于错误恢复：如果坐标在不同截图之间发生漂移，这个语义提示可以让系统重新定位目标。

### 动作 schema

一个典型的动作 schema 包含 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每一步输出一个动作。浏览器封装层执行该动作并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，没有结构信息。最通用；在任何应用上都能工作。
- 无障碍树：结构化的 DOM / iOS 无障碍信息。定位可靠性高得多；但只在该树可用的场合有效。
- 混合模式：两者都用，用树作为原子动作的可靠定位器，用截图提供语义上下文。

生产环境的智能体在可能时会使用混合模式。浏览器自动化（Selenium + 无障碍接口）总是有树可用；桌面应用则不一定。

### 长程记忆

一个 20 步的工作流会产生 20 张截图，VLM 的上下文很快就被填满。三种压缩策略：

- 摘要链（summary-chain）：每 5 步之后，总结一次已发生的事情，丢弃旧截图。
- 跳帧（skip-frame）：保留第一张、最后一张，以及每隔 3 张的截图。
- 工具记录日志：执行动作后保留一份"做了什么"的文本日志；不再回看旧截图。

Claude 的 computer-use API 采用的就是日志模式。更简单，也更可靠。

### 视觉工具调用

ChartAgent（arXiv:2510.04514）为图表理解引入了视觉工具调用：裁剪、缩放、OCR、调用外部检测。智能体可以输出"裁剪到区域 (100, 200, 300, 400)，然后调用 OCR"作为一次工具调用。工具返回文本，VLM 继续推理。

这一模式可以泛化：set-of-mark 提示、区域标注、外部检测工具，全都符合同一个"输出工具调用、接收结构化响应"的 schema。

### 2026 年的基准

- ScreenSpot-Pro。约 1k 张网页截图上的 GUI 定位。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿模型约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类信息）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最难的基准。覆盖 12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。较早的基准；已被前沿模型刷到饱和。

### 为什么这仍然很难

智能体性能的瓶颈：

1. 细粒度视觉定位。在移动端分辨率下，"点击那个小小的 X"经常失败。
2. 长程规划。执行 10 个动作之后，智能体会偏离目标。
3. 错误恢复。当一次点击失败（点错按钮）时，检测并恢复的能力几乎没有对应的训练数据。
4. 跨页面上下文。在标签页之间跳转或填写长表单时会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（用截图比对来确认动作是否成功）。

### 毕业项目的实现部分

毕业项目的任务：构建一个计算机操作（computer-use）智能体，它能：

1. 读取一个订票网站模拟页面的 HTML + 截图。
2. 规划一个多步骤序列：搜索 → 选择 → 填表 → 提交。
3. 输出符合动作 schema 的 JSON 动作。
4. 在固定的 10 个任务切片上进行评估。

本课提供的脚手架代码可以很容易地扩展到真实浏览器上。

## 生产实践

`code/main.py` 是毕业项目的脚手架：

- 动作 schema 的 JSON 定义（10 种动作）。
- 以 dict 表示的模拟浏览器状态。
- 智能体循环骨架：接收状态、输出动作、应用动作、循环。
- 10 个任务的迷你基准（合成页面），用于衡量端到端成功率。
- 动作失败时的错误恢复钩子。

## 交付产物

本课产出 `outputs/skill-multimodal-agent-designer.md`。给定一个计算机操作产品（领域、动作集合、评估目标），它能设计出完整的智能体循环、记忆策略、定位模式以及预期的基准得分。

## 练习

1. 为动作 schema 扩展一个 `screenshot_region` 工具（裁剪 + 缩放）。哪些任务会从中受益？

2. 阅读 AgentVista（arXiv:2602.23166）。描述其中最难的任务类别，以及前沿模型为何仍会失败。

3. 长程记忆压缩：设计一个摘要链，活跃截图保留不超过 4 张，日志条数不限。

4. 构建一个错误恢复钩子：当动作失败（找不到按钮）时，智能体接下来该怎么做？

5. 在 10 个网页任务上对比纯截图的 Claude 4.7 与"截图 + 无障碍树"混合模式的 Qwen2.5-VL。哪种方案在哪些任务上胜出？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| GUI 定位 | "点击坐标" | 模型针对截图上指令的目标输出 (x,y) 坐标 |
| 动作 schema | "工具定义" | 对有效动作（点击、输入、滚动、拖拽）的 JSON 描述 |
| 无障碍树 | "结构化 DOM" | 来自浏览器 / iOS API 的机器可读 UI 层级结构 |
| 混合智能体 | "截图 + 树" | 同时使用图像和结构化信息；比任何一种单独使用都更可靠 |
| 视觉工具调用 | "缩放/裁剪/检测" | 智能体在规划过程中调用外部视觉工具（OCR、检测） |
| 摘要链 | "记忆压缩" | 用周期性的文本摘要替代冗长的截图历史 |
| VisualWebArena | "端到端网页基准" | 2024 年的端到端网页任务基准 |
| AgentVista | "2026 困难基准" | 覆盖 12 个领域的真实工作流；连 Gemini 3 Pro 也只有约 30% 的得分 |
